# S1-06 — Refactor `stator.ts` — assemblage final
**Effort:** M | **Sprint:** 1 | **Dépend de:** S1-03, S1-04, S1-05

## Contexte
Après S1-02 à S1-05, `Stator.ts` ne contient plus que la factory `stator()` et ses helpers locaux (`isPrimitive`, `notify`, `setOnChange`). Cette story finalise le refactoring en : ajoutant `subscribe` et `batch` à l'objet retourné, et en s'assurant que `index.ts` exporte également `batch` comme fonction standalone.

## État attendu de `Stator.ts` après toutes les stories précédentes

À ce stade, `Stator.ts` devrait ressembler à ceci (c'est le contenu CIBLE à atteindre) :

```typescript
import type { Primitive, StateWrapper, StateChangeHandler, AugmentedState, HandlerContext } from './types.js';
import { getEventTarget } from './polyfill.js';
import { createProxy } from './proxy.js';
import { batch } from './batch.js';

export function stator<T>(initialState: T): AugmentedState<T> {

  const eventTarget = getEventTarget();
  const proxyCache = new WeakMap<object, any>();

  let onchange: StateChangeHandler<T> | undefined;
  let onchangeListener: ((e: Event) => void) | undefined;

  function isPrimitive(val: unknown): val is Primitive {
    return val === null || (typeof val !== 'object' && typeof val !== 'function');
  }

  function notify(oldValue: T, newValue: T) {
    eventTarget.triggerChange(new CustomEvent('stator:change', { detail: { oldValue, newValue } }));
  }

  function setOnChange(handler: StateChangeHandler<T> | undefined) {
    if (onchangeListener) {
      eventTarget.removeEventListener('stator:change', onchangeListener);
      onchangeListener = undefined;
    }
    onchange = handler;
    if (handler) {
      onchangeListener = (e: any) => onchange?.(e.detail.oldValue, e.detail.newValue);
      eventTarget.addEventListener('stator:change', onchangeListener);
    }
  }

  const stateWrapper: StateWrapper<T> = { _value: undefined as any };

  const ctx: HandlerContext<T> = {
    getOnChange: () => onchange,
    setOnChange,
    rootState: stateWrapper,
    eventTarget: {
      addEventListener: eventTarget.addEventListener.bind(eventTarget),
      removeEventListener: eventTarget.removeEventListener.bind(eventTarget),
      triggerChange: eventTarget.triggerChange.bind(eventTarget),
    },
    notify,
    isPrimitive,
    createDeepProxy: (obj: any) => createProxy(obj, false, ctx),
    proxyCache,
  };

  stateWrapper._value = isPrimitive(initialState) ? initialState : createProxy(initialState, false, ctx);

  // Create the root proxy
  const rootProxy = createProxy(stateWrapper, true, ctx) as unknown as AugmentedState<T>;

  // Augment with subscribe() and batch()
  // We use Object.defineProperties on the stateWrapper so the root proxy's
  // get trap forwards them correctly via the reserved properties map.
  // Simpler approach: return a wrapper object that delegates to rootProxy.

  const augmented: AugmentedState<T> = {
    get value() { return rootProxy.value; },
    set value(v: T) { (rootProxy as any).value = v; },
    get stator() { return rootProxy.stator; },
    set stator(v: T) { (rootProxy as any).stator = v; },
    get onchange() { return rootProxy.onchange; },
    set onchange(fn: StateChangeHandler<T> | undefined) { rootProxy.onchange = fn; },
    addEventListener: (...args) => rootProxy.addEventListener(...args),
    removeEventListener: (...args) => rootProxy.removeEventListener(...args),
    triggerChange: (e) => rootProxy.triggerChange(e),
    toString: () => rootProxy.toString(),
    valueOf: () => rootProxy.valueOf(),
    [Symbol.toPrimitive]: (hint: string) => (rootProxy as any)[Symbol.toPrimitive](hint),

    // New: subscribe — returns unsubscribe function
    subscribe(listener: StateChangeHandler<T>): () => void {
      const handler = (e: Event) => {
        const { oldValue, newValue } = (e as CustomEvent).detail;
        listener(newValue, oldValue);
      };
      eventTarget.addEventListener('stator:change', handler);
      return () => eventTarget.removeEventListener('stator:change', handler);
    },

    // New: batch — defer notifications
    batch(fn: () => void): void {
      batch(fn);
    },
  };

  return augmented;
}
```

## Fichier à MODIFIER : `src/lib/index.ts`

**AVANT :**
```typescript
export * from '$lib/stator/Stator.js';
```

**APRÈS :**
```typescript
export * from '$lib/stator/Stator.js';
export { batch } from '$lib/stator/batch.js';
```

Cela permet aux consommateurs d'importer `batch` directement :
```typescript
import { stator, batch } from '@medyll/idae-stator';
```

## Acceptance Criteria
- [ ] `augmented.subscribe(listener)` fonctionne et retourne une fonction unsubscribe
- [ ] `augmented.batch(fn)` délègue bien à la fonction `batch` de `batch.ts`
- [ ] `batch` est exporté depuis `index.ts` comme export standalone
- [ ] `augmented.value`, `augmented.stator`, `augmented.onchange` fonctionnent toujours
- [ ] `npm test` passe intégralement
- [ ] `npm run build` produit un `dist/` sans erreur
- [ ] `npm run check` sans erreur TypeScript

## Pièges à éviter
- L'ordre des arguments dans `subscribe` : le listener reçoit `(newValue, oldValue)` — dans `stator:change`, `detail` est `{ oldValue, newValue }`, donc inverser à l'appel du listener
- Ne pas casser l'accès proxy existant : `augmented` délègue à `rootProxy` pour `value`/`stator`/`onchange`
- Le nom `batch` est utilisé à la fois comme import de `./batch.js` et comme méthode — utiliser un alias si collision : `import { batch as batchFn } from './batch.js'`
