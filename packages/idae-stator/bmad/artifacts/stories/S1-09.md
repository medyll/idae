# S1-09 — Vérifier couverture ≥ 90% + fix gaps
**Effort:** S | **Sprint:** 1 | **Dépend de:** S1-08

## Contexte
Configurer Vitest pour mesurer la couverture de code et s'assurer qu'elle atteint ≥ 90% sur `src/lib/stator/`. Ajouter les tests manquants si nécessaire.

## Étape 1 : Configurer la couverture dans `vite.config.ts`

Fichier : `vite.config.ts` (à la racine du projet)

Lire d'abord le fichier actuel, puis ajouter la configuration `test.coverage`. Le fichier ressemble probablement à :

```typescript
import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [sveltekit()],
  // ... éventuellement une section test existante
});
```

Ajouter ou compléter la section `test` :

```typescript
import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [sveltekit()],
  test: {
    environment: 'jsdom',
    coverage: {
      provider: 'v8',
      include: ['src/lib/stator/**/*.ts'],
      exclude: ['src/lib/stator/**/*.test.ts'],
      thresholds: {
        lines: 90,
        functions: 90,
        branches: 90,
        statements: 90,
      },
      reporter: ['text', 'html'],
    },
  },
});
```

**Note :** si `environment: 'jsdom'` est déjà présent, ne pas le dupliquer.

## Étape 2 : Installer `@vitest/coverage-v8` si nécessaire

Vérifier dans `package.json` si `@vitest/coverage-v8` est déjà dans `devDependencies`. Si non, l'ajouter :

```bash
npm install --save-dev @vitest/coverage-v8
```

## Étape 3 : Lancer la couverture

```bash
npm run test -- --coverage
```

ou directement :
```bash
npx vitest run --coverage
```

## Étape 5 : Gaps courants à couvrir

### `polyfill.ts` — branche Node.js
```typescript
it('creates EventTarget in Node.js environment (no window/document)', () => {
  // Ce test est déjà couvert par les tests existants qui tournent en jsdom
  // Pour couvrir la branche Node.js, ajouter un test avec vi.stubGlobal
  const originalWindow = global.window;
  const originalDocument = global.document;

  vi.stubGlobal('window', undefined);
  vi.stubGlobal('document', undefined);

  const { getEventTarget } = await import('./polyfill.js');
  const et = getEventTarget();
  const fn = vi.fn();
  et.addEventListener('test', fn);
  et.triggerChange(new Event('test'));
  expect(fn).toHaveBeenCalledTimes(1);

  vi.unstubAllGlobals();
});
```

### `proxy.ts` — branche `deleteProperty`
```typescript
it('deleteProperty triggers notification', () => {
  const state = stator<Record<string, number>>({ a: 1, b: 2 });
  const fn = vi.fn();
  state.subscribe(fn);
  delete (state.value as any).a;
  expect(fn).toHaveBeenCalledTimes(1);

```

### `proxy.ts` — branche primitive set returns false
```typescript

  const state = stator({ num: 42 });
  // num is primitive, accessing it creates no sub-proxy
```

- [ ] Le rapport HTML est généré dans `coverage/` (gitignorer ce dossier)

## Pièges à éviter
- Si `vitest` échoue avec "coverage provider not found", installer `@vitest/coverage-v8`
- Les branches TypeScript non-exécutables (ex: `never`) ne comptent pas — se concentrer sur les branches runtime
- Ne pas viser 100% : 90% est le seuil, au-delà c'est du temps perdu sur des cas non-critiques
- `types.ts` ne contient que des types — sa couverture sera 100% automatiquement (pas de code runtime)
