# S1-01 — Corriger `package.json` — deps runtime
**Effort:** XS | **Sprint:** 1 | **Dépend de:** —

## Contexte
Le fichier `package.json` déclare `commander` et `command` dans `dependencies`, ce qui signifie qu'ils sont installés chez les consommateurs de la lib. Ce sont des outils CLI uniquement — ils ne doivent pas être dans le bundle. De plus, `svelte` est une peer dep mais n'est pas marquée comme optionnelle.

## Fichier à modifier
`D:\boulot\dev\node\idae\packages\idae-stator\package.json`

## Changements exacts à effectuer

### 1. Déplacer `commander` et `command` de `dependencies` vers `devDependencies`

**AVANT** (section `dependencies`) :
```json
"dependencies": {
  "command": "^1.1.1",
  "commander": "^14.0.3"
}
```

**APRÈS** : supprimer complètement la section `dependencies` (elle sera vide) :
```json
"dependencies": {}
```

Et ajouter dans `devDependencies` (qui existe déjà) les deux entrées :
```json
"command": "^1.1.1",
"commander": "^14.0.3",
```

### 2. Ajouter `peerDependenciesMeta` après `peerDependencies`

**AVANT** :
```json
"peerDependencies": {
  "svelte": "^5.0.0-next"
},
```

**APRÈS** :
```json
"peerDependencies": {
  "svelte": "^5.0.0-next"
},
"peerDependenciesMeta": {
  "svelte": {
    "optional": true
  }
},
```

## Acceptance Criteria
- [ ] La section `dependencies` est vide `{}` ou absente
- [ ] `commander` et `command` sont présents dans `devDependencies`
- [ ] `peerDependenciesMeta` est présent avec `svelte: { optional: true }`
- [ ] Le JSON est valide (pas de virgule manquante ou en trop)
- [ ] Commande `npm run build` passe sans erreur (ne pas oublier de vérifier)

## Pièges à éviter
- Ne pas supprimer d'autres entrées dans `devDependencies`
- Ne pas toucher aux `scripts`, `exports`, ou autres sections
- Vérifier que le JSON reste syntaxiquement valide après modification
