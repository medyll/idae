# Audit Report — idae-stator
**Date:** 2026-03-17
**Auditor:** bmad [Reviewer]
**Version:** 0.51.2

---

## Codebase Health Score: 7.5 / 10

Small, focused library with a clean public API and a solid test suite. The core Proxy-based state management is well-structured. Main concerns are known limitations not surfaced to consumers, `any` usage in tests, and indirect dependency vulnerabilities.

---

## Security Scan

All vulnerabilities are **indirect** (transitive dependencies of other workspace packages, not idae-stator itself). idae-stator has `"dependencies": {}` — zero runtime deps — so none of these affect published consumers.

| Severity | Package | Issue | Path |
|----------|---------|-------|------|
| Critical | `form-data <2.5.4` | Unsafe random boundary | `idae-socket > request > form-data` |
| Critical | `basic-ftp <5.2.0` | Path traversal in `downloadToDir()` | `idae-dom-events > puppeteer > ... > basic-ftp` |
| High | `node-fetch` | Forwards secure headers to untrusted sites | indirect |
| Moderate | `fast-xml-parser >=5.0.0 <5.3.5` | ReDoS in DOCTYPE entity names | indirect |

**Action:** These need to be fixed in the owning workspace packages (`idae-socket`, `idae-dom-events`, `idae-db`), not here.

## Code Quality

### Critical
_None._

### Important

1. **Known Proxy limitations not exposed in public API** (`Stator.ts`)
   `Map`, `Set`, `RegExp` stored in state silently fail — methods like `map.get()` throw at runtime through the proxy. These are documented in tests but not in the public `AugmentedState` type or JSDoc.
   → Add a `@remarks` / warning in the exported type or README listing unsupported built-ins.

2. **`peerDependencies` uses outdated range** (`package.json`)
   `"svelte": "^5.0.0-next"` — Svelte 5 stable was released. Should be `"^5.0.0"`.

3. **Test files included in `dist/`**
   `files` array excludes `dist/**/*.test.*` but tests are being compiled into `.svelte-kit/__package__/` and `dist/` (confirmed by glob). The `prepackage` script may not be cleaning these reliably.
   → Verify `scripts/package-pre.js` removes test artifacts before publish.

### Minor

4. **Pervasive `any` in test helpers** (`Stator.test.ts`)
   `deepFind(obj: any)`, `deepFindArrayWith4(obj: any)`, `deepFindB3(obj: any)` — inline recursive helpers are duplicated ~4 times with nearly identical structure. If these grow, extract a single typed `deepFind<T>(obj: unknown, predicate: (v: unknown) => boolean): boolean` utility.

5. **`peerDependencies` not listed in `devDependencies` correctly**
   `svelte` is in devDependencies as `"^5.50.0"` but peer is `"^5.0.0-next"` — the peer constraint is stale.

6. **`command` devDependency appears unused** (`package.json`)
   Both `command` and `commander` are listed. `command ^1.1.1` is almost certainly a leftover.
