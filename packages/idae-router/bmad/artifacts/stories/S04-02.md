# Story S04-02 – Add JSDoc to all public exports

**Epic:** Type-Safety / Developer Experience  
**Sprint:** 04  
**Points:** 2  
**Priority:** Must  
**PRD refs:** FR-B04, FR-B05, FR-B06, FR-B08, FR-F08, FR-F10  
**Status:** ✅ Implemented (2026-03-02)  
**Depends on:** S04-01

---

## User Story

As a **developer consuming `idae-router`**, I want every public function, class, and type to have complete JSDoc annotations so that **IDE hover, `tsc` feedback, and generated API docs all reflect the real API contract**.

---

## Context

Covers all files in `src/lib/`: `router.ts`, `fetcher.ts`, `matcher.ts`, `render.ts`, and the new `types.ts` from S04-01. JSDoc must use standard tags recognised by TypeScript's `@link`, `@param`, `@returns`, `@throws`, `@example`, `@since`, `@public`, `@see`.

Already-documented symbols (`fetcher.ts`) are a reference baseline; this story brings every other public symbol up to the same standard.

---

## Acceptance Criteria

```gherkin
Given any exported function in src/lib/
When a developer hovers over it in VS Code
Then @param, @returns, and @example tags appear in the tooltip

Given router.push, router.replace, router.prefetch, router.invalidate, router.buildUrl
When the developer inspects the type
Then @param, @returns, @example, and @since tags are all present

Given router.onError callback
When the developer hovers
Then a @example showing error-handler setup is visible

Given all exported interface members (Route, Context, RouterOptions, RouterInstance)
When a developer hovers over any property
Then an inline JSDoc comment is visible
```
