# Story S01-04 – Add test coverage reporting and expand test suite

**Epic:** Testing  
**Sprint:** 01  
**Points:** 1  
**Priority:** Should

## User Story

As a **project maintainer**, I want to **measure and track code coverage**, so that **I can identify untested paths and prevent regressions in critical router logic**.

---

## Context

The idae-router test suite covers core functionality (matcher, router, render) but lacks:
1. Coverage reporting infrastructure
2. Baseline coverage measurement
3. Tests for hash mode routes
4. Tests for 404 fallback behavior
5. Tests for error boundary scenarios

**Current State:**
- Unit tests: 7 (matcher + router)
- E2E tests: 4 (Playwright)
- Coverage reporting: ❌ None
- Coverage target: Not set

**Problem:** Without coverage reports, developers cannot:
- See which lines are untested
- Track coverage improvements over time
- Prevent feature regressions
- Identify risky code paths

**Scope (Sprint 01):**
1. Add `@vitest/coverage-v8` to devDependencies
2. Configure `vite.config.ts` with coverage settings
3. Run baseline coverage report
4. Document coverage expectations (80%+ for Sprint 02+)

**Scope (Future - Sprint 02+):**
- Add tests for hash mode, 404 routes, error boundaries
- Achieve 85%+ coverage

**Out of Scope:**
- End-to-end coverage (that's app-level responsibility)
- Performance benchmarks — separate story

## Technical Notes

### Current Test Setup

**vitest config** in `vite.config.ts`:
```typescript
test: {
  globals: true,
  environment: 'jsdom',
}
```

### Missing Coverage Config

Need to add:
```typescript
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,
    environment: 'jsdom',
    // NEW: Coverage config
    coverage: {
      provider: 'v8',
      reporter: ['text', 'json', 'html'],
      exclude: [
        'node_modules/',
        'dist/',
        'src/**/*.test.ts',
        'src/**/*.spec.ts',
      ],
      lines: 75,      // minimum line coverage
      functions: 75,
      branches: 70,
      statements: 75,
    },
  },
});
```

### Implementation Strategy

1. **Install coverage provider**:
   ```bash
   npm install --save-dev @vitest/coverage-v8
   ```

2. **Update `vite.config.ts`** with coverage config (see above)

3. **Run baseline coverage**:
   ```bash
   npm run test:unit -- --coverage
   ```
   This generates:
   - `coverage/index.html` (visual report)
   - `coverage/coverage-final.json` (machine-readable)

4. **Add npm script** (optional, for convenience):
   ```json
   {
     "test:coverage": "vitest run --coverage"
   }
   ```

5. **Document coverage expectations**:
   - Update `README.md` or `SPECS.md` with coverage target (75%+ in Sprint 01, 85%+ in Sprint 02)
   - Add link to coverage report in CI/CD

6. **Optional: Add CI step** to fail if coverage drops below threshold:
   ```yaml
   # In CI config (e.g., GitHub Actions)
   - run: npm run test:coverage
     # CI fails if coverage < 75%
   ```
