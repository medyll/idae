# Story S02-03 – Validate npm package and publish v0.1.0

**Epic:** Publishing  
**Sprint:** 02  
**Points:** 2  
**Priority:** Should  
**Status:** Ready for Development (after Sprint 01 merge + S02-01/02 complete)

---

## User Story

As a **package maintainer**, I want to **publish @medyll/idae-router to the npm registry**, so that **developers can install and use the router as a public library**.

---

## Context

The `@medyll/idae-router` package is production-ready after Sprint 01 (security fixes, packaging, error handling) and Sprint 02 (expanded test coverage). Now it's time to publish v0.1.0 to npm.

**Current State:**
- Version in `package.json`: 0.0.1
- Scope: @medyll (public npm access configured)
- Package built and validated with `publint`
- No CHANGELOG yet

**Problem:** Without publishing, the library isn't accessible to end users.

**Scope:**
1. Update version to v0.1.0 in `package.json`
2. Create CHANGELOG.md documenting all changes
3. Run final package validation (`npm pack`, `publint`)
4. Authenticate with npm registry
5. Publish package to npm
6. Verify package on npm registry
7. Update README with npm install link

**Out of Scope:**
- Version bumping strategy (semver to be defined in future releases)
- Release notes distribution (GitHub releases, announcements)
- npm organization setup (already done)

# Changelog – @medyll/idae-router

All notable changes to this project are documented here. Follows [Semantic Versioning](https://semver.org).

## [0.1.0] — 2026-03-09

### Added
- XSS vulnerability mitigation: HTML sanitization in route content (S01-01)
- CommonJS export support via `main` field in package.json (S01-02)
- Error handler callback: optional `onError` on RouterInstance (S01-03)

- Hash mode routing tests (S02-01)
- 404 fallback behavior tests (S02-02)

### Fixed
- Silent Promise.catch blocks now log errors (S01-03)
- Test files excluded from npm package (S01-02)

### Changed
- Enhanced error handling throughout navigation pipeline

### Security
- **CRITICAL**: Fixed XSS vulnerability in route content rendering

## [0.0.1] — Initial release
- Basic SPA routing with history and hash modes
- Route parameters, query strings, nested routes
- Before/after navigation hooks
- Link interception

### Publishing Checklist

| Step | Command | Expected Result |
|------|---------|-----------------|
| 1. Verify build | `npm run build` | No errors, `dist/` generated |
| 2. Package validation | `npm pack` | Tarball created, no .test files |
| 3. Final lint | `publint` | 0 errors, 0 warnings |
| 4. npm login | `npm login` | Authenticated to npm |
| 5. Dry run | `npm publish --dry-run` | Shows what would be published |
| 6. Publish | `npm publish` | Package published, no errors |
| 7. Verify registry | Visit npm.js | Package visible, version correct |
| 8. Test install | `npm install @medyll/idae-router@0.1.0` | Installation successful |

## Acceptance Criteria

- [ ] **AC1: Version updated**
  - `package.json` version changed to 0.1.0
  - No other version strings need manual update (git tags are manual)

- [ ] **AC2: CHANGELOG.md created**
  - File at `CHANGELOG.md` in repo root
  - Documents all S01 and S02 changes
  - Follows SemVer format
  - Security fix highlighted

- [ ] **AC3: Package validation passed**
  - `npm pack` creates tarball without errors
  - No `.test.js` or `.spec.js` files in tarball
  - `publint` passes with 0 errors, 0 warnings

- [ ] **AC4: npm authentication**
  - `npm login` succeeds (credentials in .npmrc)
  - `npm whoami` shows correct user/org

- [ ] **AC5: Package published**
  - `npm publish` succeeds
  - No errors about versioning or metadata
  - Build artifacts uploaded to registry

- [ ] **AC6: Package discoverable on npm**
  - Visit https://www.npmjs.com/package/@medyll/idae-router
  - v0.1.0 listed as latest version
  - README renders correctly
  - All metadata visible (keywords, repository, author)

- [ ] **AC7: Installation works**
  - `npm install @medyll/idae-router@0.1.0` succeeds in a test project
  - Package can be imported: `import { createRouter } from '@medyll/idae-router'`
  - Types available (TypeScript projects)

- [ ] **AC8: Documentation updated**
  - README.md updated with npm install instructions
  - Link to npm package added
  - Version number referenced in docs
