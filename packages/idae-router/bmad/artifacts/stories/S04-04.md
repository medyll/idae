## User Story

As a **library internals developer**, I want a standalone `cache.ts` module so that **the router can serve stale data immediately and revalidate in the background**, eliminating perceived fetch latency on back-navigation without introducing any external dependency.

---

## Context

Cache is an in-memory `Map`. No `localStorage` persistence. Strategy is stale-while-revalidate (SWR). The cache is consumed exclusively by `router.ts` (S04-04). This module is tree-shakeable: if `cache: false` globally, the import of `cache.ts` is eliminated by the bundler.

---

## Acceptance Criteria

```gherkin
Given a route fetch that resolves with data for URL '/api/users/42'
When fetchAndCache is called
Then the result is stored in the cache keyed by 'GET /api/users/42'

Given an entry stored less than ttl ms ago
When getCacheEntry is called for that key
Then the entry is returned (cache hit)

Given an entry stored more than ttl ms ago
When getCacheEntry is called for that key
Then undefined is returned (TTL expired, entry evicted)

Given an entry stored less than staleTime ms ago
When getCacheEntry is called
Then state 'fresh' is returned (no revalidation needed)

Given an entry stored between staleTime and ttl ms ago
When getCacheEntry is called
Then state 'stale' is returned (serve + revalidate)

Given createRouter({ cache: false })
When the cache module API is called
Then no reads or writes occur (no-op mode)

Given invalidate('/users/*') is called
When getCacheEntry is called for '/api/users/42'
Then undefined is returned (entry cleared by glob)

Given the same data is returned from revalidation
When deep-equal comparison runs
Then action is NOT re-invoked
```

Cache key formula: `"${method} ${resolvedUrl}"` (e.g. `"GET /api/users/42"`)

Glob matching for `invalidate`:
- Exact string: `invalidate('/api/users/42')` clears one entry
- Glob `*` at end: `invalidate('/api/users/*')` clears all entries whose key path starts with `/api/users/`
- Use simple startsWith logic — no external glob library.
