# Story S01-02 – Add CommonJS export to package.json

**Epic:** Packaging  
**Sprint:** 01  
**Points:** 2  
**Priority:** Must  
**Status:** Ready for Development

---

## User Story

As a **Node.js package consumer**, I want to **require the idae-router package in CommonJS environments**, so that **my CJS/Node.js application can use the router without ES module errors**.

---

## Context

Currently, `package.json` only exports ESM (Svelte field), blocking CommonJS consumers:

```json
{
  "svelte": "./dist/index.js",
  "types": "./dist/index.d.ts"
}
```

**Problem:** CommonJS apps using `require('@medyll/idae-router')` receive "Cannot find module" or unclear errors because the `main` field is missing.

**Impact:** Broader ecosystem compatibility; unblocks:
- Legacy Node.js projects using CJS
- Bundlers that default to CJS resolution
- Monorepo workspaces using CJS module format

**Scope:**
1. Add `"main": "./dist/index.js"` to `package.json`
2. Fix `files` glob to exclude test files from npm package
3. Verify `npm pack` excludes `.test.js` and `.spec.js` files
4. Run `publint` validation to ensure no warnings

**Out of Scope:**
- Dual package hazard (ESM + CJS transpilation) — this is ESM-only, `main` field just points to the ESM entry
- Building separate CJS output — not needed; Node.js ESM is standard now

---

## Technical Notes

### Current package.json

```json
{
  "name": "@medyll/idae-router",
  "version": "0.0.1",
  "svelte": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "type": "module",
  "files": [
    "dist",
    "!dist/**/*.test.*",
    "!dist/**/*.spec.*"
  ]
}
```

### Issues to Fix

1. **Missing `main` field** — CommonJS resolution fails
2. **Test files still in dist/** — Despite `files` exclusion, `svelte-package` includes test files
3. **Need to verify `publint` passes** — npm package validator

### Implementation Strategy

1. **Update `package.json`**:
   ```json
   {
     "main": "./dist/index.js",
     "svelte": "./dist/index.js",
     "types": "./dist/index.d.ts",
     "type": "module",
     "files": ["dist", "!dist/**/*.{test,spec}.*"]
   }
   ```

2. **Check svelte.config.js** for any package-level config:
   - Verify SvelteKit sync doesn't include test files
   - If needed, add exclude pattern

3. **Run build & validation**:
   ```bash
   npm run build    # runs: vite build && npm run prepack
   npm pack         # create tarball
   # Inspect tarball contents to confirm no .test.js or .spec.js files
   publint          # validate package metadata
   ```

4. **Test with CJS consumer** (optional, for verification):
   ```javascript
   const router = require('@medyll/idae-router');
   console.log(typeof router.createRouter); // should print 'function'
   ```

## Implementation Notes

**Date:** 2026-03-02  
**Status:** Implemented (pending code review & testing)

**Files changed:**
- `package.json` — Added `"main": "./dist/index.js"` field + updated `files` glob to `["dist", "!dist/**/*.{test,spec}.*"]`
- (No source code changes)

**Key changes:**
1. Added CommonJS default export via `main` field
2. Fixed `files` glob to use `{test,spec}` pattern (cleaner than separate patterns)
3. Maintains ESM as primary (via `svelte` field)

**Notable decisions:**
- `main` field points to the same `dist/index.js` as `svelte` (ESM-only, no dual CJS/ESM transpilation)
- Node.js 16+ supports ESM via `"type": "module"`, so no need for separate CJS output
- This expands compatibility with bundlers and monorepos that check the `main` field

**Known limitations:**
- CommonJS `require()` will get ESM, which Node.js handles via CJS-ESM interop (works for named exports)
- Very old CommonJS-only environments (Node <14) will not work, but that's end-of-life anyway
