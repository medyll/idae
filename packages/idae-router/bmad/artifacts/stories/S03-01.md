# Story S03-01 – Extend `Route` and `Context` types for HTTP data-fetching

**Epic:** Types  
**Sprint:** 03  
**Points:** 1  
**Priority:** Must  
**Status:** pending  
**PRD refs:** FR-01, FR-02, FR-06, FR-07, FR-08

---

## User Story

As a **library consumer**, I want `Route` to accept `http` and `http_source` options with full TypeScript types, so that **my IDE gives autocompletion and type-safety when configuring route-level data fetching**.

---

## Context

`src/lib/types.ts` defines `Route` and `Context`. Neither currently has data-fetching fields. This story adds the new types before any implementation touches `router.ts` or `fetcher.ts`, ensuring downstream stories compile against a stable contract.

---

## Acceptance Criteria

```gherkin
Given I define a route with http: { url: '/api/users/:id' }
When I run tsc --noEmit
Then there are no type errors

Given I define a route with http_source: { url: 'api.example.com/users/:id', args: { method: 'GET' } }
When I run tsc --noEmit
Then there are no type errors

Given a route action receives ctx
When I access ctx.data and ctx.error
Then TypeScript resolves them as unknown and Error | undefined respectively

Given a consumer imports RouteHttpConfig from '@medyll/idae-router'
Then the type is available in their project
```

## Technical Notes

**File to edit:** `src/lib/types.ts`

Add after the existing `Action` type:

```typescript
/**
 * Configuration for a route-level HTTP data-fetch.
 *
 * - For `http`: provide a path-only URL (e.g. `/api/users/:id`).
 *   `window.location.origin` is prepended automatically.
 * - For `http_source`: provide host+path without scheme (e.g. `api.example.com/users/:id`).
 *   `https://` is prepended automatically.
 *
 * `:param` tokens are interpolated from matched route params before fetching.
 */
export interface RouteHttpConfig {
  url: string;
  args?: RequestInit;
}
```

Extend `Route`:

```typescript
export interface Route {
  path: string;
  action?: Action;           // was required — make optional (action OR http/http_source suffices)
  metadata?: Record<string, unknown>;
  children?: Route[];
  http?: RouteHttpConfig;
  http_source?: RouteHttpConfig;
}
```

Extend `Context`:

```typescript
export interface Context {
  path: string;
  params: Record<string, string>;
  query: Record<string, string | string[]>;
  state?: unknown;
  metadata?: Record<string, unknown>;
  matched?: RouteRecord[];
  /** Populated by `http` or `http_source` before `action` executes. null on fetch error. */
  data?: unknown;
  /** Set when the route-level fetch fails. Navigation continues regardless. */
  error?: Error;
}
```

**File to edit:** `src/lib/index.ts`  
Export `RouteHttpConfig` so consumers can type their own configs.
