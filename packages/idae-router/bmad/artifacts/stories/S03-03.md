# Story S03-03 – Integrate fetcher into `handleNavigation` in `router.ts`

**Epic:** Core  
**Sprint:** 03  
**Points:** 2  
**Priority:** Must  
**Status:** pending  
**PRD refs:** FR-06, FR-07, FR-11, UC-01, UC-02, UC-04

---

## User Story

As an **app developer**, I want `ctx.data` to be populated automatically before my route `action` runs, so that **I can use fetched data in my render function without writing any fetch boilerplate**.

---

## Context

`handleNavigation` in `src/lib/router.ts` already iterates over the matched route chain and calls each route's `action`. This story injects a `fetchRouteData` call per route level (before the `action` call), enriching the shared `Context` object with `data` and `error`.

The enrichment happens **per route level in the chain**, so a parent route with `http` and a child route with `http_source` both get their own independent fetch. Context is cloned or extended per level — the parent's `ctx.data` is isolated from the child's.

---

## Acceptance Criteria

```gherkin
Given a route with http: { url: '/api/users/:id' } and a matched param id='42'
When the user navigates to /users/42
Then ctx.data inside action() equals the JSON returned by /api/users/42

Given a route with http_source: { url: 'api.example.com/items/1' }
When the user navigates to the route
Then ctx.data inside action() equals the JSON from https://api.example.com/items/1

Given fetch fails with a network error
When the user navigates to the route
Then ctx.error is an Error instance AND ctx.data is null AND navigation completes (action still runs)

Given a route with no http or http_source
When the user navigates to the route
Then ctx.data and ctx.error remain undefined (no fetch is executed)

Given a parent route with http and a child route with http_source
When the user navigates to the child path
Then parent action receives parent ctx.data, child action receives child ctx.data independently
```

## Technical Notes

**File to edit:** `src/lib/router.ts`

Import `fetchRouteData` from `./fetcher.js`.

In `handleNavigation`, inside the per-level `for (const rec of chain)` loop, before calling `action`:

```typescript
// Before: const result = await Promise.resolve(action(toContext));
// After:

// Enrich context with route-level fetch data
if (rec.route.http || rec.route.http_source) {
  const { data, error } = await fetchRouteData(rec.route, toContext.params);
  // Extend context for this level only (do not mutate the shared toContext)
  levelContext = { ...toContext, data, error };
} else {
  levelContext = toContext;
}

const action = rec.route.action;
if (action) {
  const result = await Promise.resolve(action(levelContext));
  // ...existing mount logic...
}
```

> **Note on action optionality:** Now that `action` is optional (S03-01 makes it `action?: Action`), add a guard: if `!action` and `http`/`http_source` defined, skip rendering for that level (or use the JSON `<pre>` fallback introduced in Sprint 04's S03-06).

**No change required to:** `matcher.ts`, `render.ts`, `buildContext`.
