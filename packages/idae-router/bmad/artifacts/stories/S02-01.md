## User Story

As a **router library maintainer**, I want to **ensure hash mode routing works correctly**, so that **hash-based SPA navigation is fully supported and tested for production use**.

---

## Context

Hash mode (`mode: 'hash'`) allows single-page applications to use URL fragments (`#/path`) for client-side routing, which is useful for:
- Older browsers without History API support
- Static hosting (GitHub Pages, Netlify)
- Legacy apps requiring hash-based navigation

**Current State:**
- Hash mode implemented in `src/lib/router.ts` (lines 38-42)
- E2E tests cover basic hash navigation (see `e2e/demo.test.ts`)
- **Missing**: Comprehensive unit tests for hash-specific behavior

**Problem:** Without thorough hash mode testing, edge cases (history back/forward, query params in hash, nested routes) may break in production.

**Scope:**
1. Add unit tests for hash mode initialization
2. Test hash route parsing (extract path and query params from `#/path?query=value`)
3. Test history back/forward with hash mode
4. Test route params in hash paths
5. Test mixed hash + query string scenarios

**Out of Scope:**
- Browser compatibility (that's E2E + CI)
- Hash collision detection (hash mode limitation, documented)
- URL encoding edge cases (handled by native APIs)

## Technical Notes

### Hash Mode Implementation

**File:** `src/lib/router.ts:37-43`

```typescript
function getPathFromLocation(): string {
  if (mode === 'hash') {
    const h = location.hash || '#/';
    return h.startsWith('#') ? h.slice(1) : h;
  }
  return location.pathname + location.search;
}
```

### Test Scenarios

| Scenario | Input | Expected Output | Test Type |
|----------|-------|-----------------|-----------|
| Basic hash navigation | `location.hash = '#/users'` | path = `/users` | Unit |
| Hash with query | `#/users?id=1&name=john` | path = `/users`, query = `{id: '1', name: 'john'}` | Unit |
| Nested hash route | `#/users/123/profile` | path = `/users/123/profile`, params = `{id: '123'}` | Unit |
| Empty hash | `location.hash = ''` | path = `/` | Unit |
| History back/forward | User clicks back after nav | Router navigates to previous hash | E2E |
| Hash with fragment | `#/page#section` | Handled by browser (fragment ignored) | Unit |

### Implementation Strategy

1. **Set up hash mode router in tests**:
   ```typescript
   const router = createRouter({
     mode: 'hash',
     routes: [...],
     outlet: document.body
   });
   ```

2. **Mock `location.hash`** in tests using `jsdom`:
   ```typescript
   // In setup or per-test
   Object.defineProperty(window.location, 'hash', {
     writable: true,
     value: '#/test'
   });
   ```

3. **Add unit test cases** for:
   - Hash path parsing
   - Query param extraction from hash
   - Route matching with hash paths
   - Context building with hash mode
   - Cleanup after navigation

4. **Add E2E test** (optional, if browser testing preferred):
   - Navigate via `location.hash = ...`
   - Verify content renders
   - Test browser back/forward button
