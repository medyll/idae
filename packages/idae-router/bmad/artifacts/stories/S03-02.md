# Story S03-02 – Create `src/lib/fetcher.ts` (URL resolution + fetch execution)

## User Story

As a **router internals maintainer**, I want a standalone `fetcher.ts` module that resolves URLs and executes fetch calls, so that **`router.ts` delegates all HTTP concerns without mixing IO logic into navigation flow**.

---

## Context

Per the PRD's implementation notes, fetch logic lives in a new `src/lib/fetcher.ts`. It is called from `handleNavigation` (S03-03) once a route with `http`/`http_source` is matched. Keeping it separate makes it independently testable and replaceable.

---

## Acceptance Criteria

```gherkin
Given a RouteHttpConfig { url: '/api/users/:id' } and params { id: '42' }
When resolveUrl is called with type 'internal'
Then it returns 'http://localhost/api/users/42' (origin prepended, :id replaced)

Given a RouteHttpConfig { url: 'api.example.com/items/:slug' } and params { slug: 'foo' }
When resolveUrl is called with type 'external'
Then it returns 'https://api.example.com/items/foo'

Given fetch resolves with a valid JSON body
When fetchRouteData is called
Then it returns { data: <parsed JSON>, error: undefined }

Given fetch rejects (network error)
When fetchRouteData is called
Then it returns { data: null, error: <Error instance> }

Given fetch resolves with a non-OK status (e.g. 404)
When fetchRouteData is called
Then it returns { data: null, error: Error('HTTP 404') }

Given a route has both http and http_source defined
When fetchRouteData is called
Then it logs a console.warn and uses http (http wins)
```

## Technical Notes

**New file:** `src/lib/fetcher.ts`

```typescript
import type { RouteHttpConfig, Context } from './types.js';

export interface FetchResult {
  data: unknown;
  error: Error | undefined;
}

/**
 * Interpolate :param tokens in a URL template using ctx.params.
 */
export function interpolateParams(url: string, params: Record<string, string>): string {
  return url.replace(/:([a-zA-Z_][a-zA-Z0-9_]*)/g, (_, key) => params[key] ?? `:${key}`);
}

/**
 * Build the full fetch URL from a RouteHttpConfig.
 * - 'internal': prepends window.location.origin (same-origin)
 * - 'external': prepends 'https://' (cross-origin, HTTPS enforced)
 */
export function resolveUrl(
  config: RouteHttpConfig,
  type: 'internal' | 'external',
  params: Record<string, string>
): string {
  const interpolated = interpolateParams(config.url, params);
  if (type === 'internal') {
    // Origin pinned — cannot be used cross-origin
    const origin = typeof window !== 'undefined' ? window.location.origin : 'http://localhost';
    return `${origin}${interpolated.startsWith('/') ? '' : '/'}${interpolated}`;
  }
  // External: HTTPS enforced
  return `https://${interpolated}`;
}

/**
 * Execute a route-level fetch and return a structured result.
 * Never throws — errors are captured in FetchResult.error.
 */
export async function fetchRouteData(
  route: { http?: RouteHttpConfig; http_source?: RouteHttpConfig },
  params: Record<string, string>
): Promise<FetchResult> {
  let config: RouteHttpConfig;
  let type: 'internal' | 'external';

  if (route.http && route.http_source) {
    console.warn('[idae-router] Route defines both `http` and `http_source`. `http` takes precedence.');
  }

  if (route.http) {
    config = route.http;
    type = 'internal';
  } else if (route.http_source) {
    config = route.http_source;
    type = 'external';
  } else {
    return { data: undefined, error: undefined };
  }

  const url = resolveUrl(config, type, params);

  try {
    const response = await fetch(url, config.args);
    if (!response.ok) {
      return { data: null, error: new Error(`HTTP ${response.status}`) };
    }
    const data: unknown = await response.json();
    return { data, error: undefined };
  } catch (err) {
    return { data: null, error: err instanceof Error ? err : new Error(String(err)) };
  }
}
```
