# Story S04-01 – Upgrade `types.ts` with generics: `Route<TData>`, `Context<TData>`

**Epic:** Type-Safety  
**Sprint:** 04  
**Points:** 2  
**Priority:** Must  
**Status:** ✅ Implemented (2026-03-02)  
**PRD refs:** FR-B01, FR-B02, FR-B03, FR-B04, FR-B05, FR-B07, FR-B08

---

## User Story

As a **library consumer**, I want `Route<TData>` and `Context<TData>` to be properly generic so that **`ctx.data` is typed as `TData | null` instead of `unknown`** inside my `action` callbacks, with full JSDoc visible on hover.

---

## Context

All other Sprint 04 features depend on the new generics. This story must land first. It is a purely additive refactor — existing routes with no `TData` specified remain `Route<unknown>` by default (backward-compatible).

---

## Acceptance Criteria

```gherkin
Given a Route<User> with http config
When the developer writes an action callback
Then ctx.data resolves as User | null (not unknown)
And ctx.error resolves as Error | undefined

Given an existing Route with only action and no generic
When TypeScript compiles the file
Then no error is emitted (defaults to Route<unknown>)

Given any public type (Route, Context, RouteHttpConfig, RouterOptions, RouterInstance)
When the developer hovers over it in their IDE
Then JSDoc with @public, @param, @returns, @example, @since is visible

Given Context<TData>
When TData is specified on the Route
Then ctx.isLoading and ctx.isRevalidating are present with boolean type
And ctx.queryError is present with Error | undefined type
```

---

## Technical Notes

**File:** `src/lib/types.ts` — full replacement

Key changes:
- `RouteHttpConfig<TData = unknown>` — add generic (url + args unchanged)
- `Route<TData = unknown>` — generic propagates to `http`, `http_source`, `action`
- `Context<TData = unknown>` — `data: TData | null` replaces `data?: unknown`; add `isLoading`, `isRevalidating`, `queryError`, `CacheOptions`
- `Action<TData = unknown>` — `(ctx: Context<TData>) => ActionResult`
- `CacheOptions` interface added (ttl, staleTime, key)
- All public symbols get full JSDoc (`@public`, `@param`, `@returns`, `@example`, `@since`)
- `RouterInstance`: extend with `prefetch`, `invalidate`, `buildUrl`, `getState`, `onError` — all JSDoc-documented

**Backward compatibility contract:**
- All existing call sites pass `Route` / `Context` without generic → must still compile as-is.
- `ctx.data` is now non-optional (`data: TData | null`). Any consumer using `ctx.data` with only an `action` (no `http`/`http_source`) gets `null` — no breaking change in behavior.
