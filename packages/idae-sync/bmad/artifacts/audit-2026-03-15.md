## Summary
**Phase:** completed | **Next action:** design-sync-outbox-api

## 🔴 Critical Issues

### 1. Duplicate OutboxStore definitions
Two parallel implementations exist and are not consolidated:
- [src/lib/OutboxStore.ts](../../src/lib/OutboxStore.ts) — top-level (legacy interface)
- [src/lib/outbox/OutboxStore.ts](../../src/lib/outbox/OutboxStore.ts) — canonical (used by all active code)

`SyncAdapter`, `Deliverer`, and `initSync` all import from `outbox/OutboxStore.ts`, making the top-level `OutboxStore.ts` dead code. **Should be removed.**

Same pattern for `InMemoryOutboxStore`:
- [src/lib/impl/InMemoryOutboxStore.ts](../../src/lib/impl/InMemoryOutboxStore.ts)
- [src/lib/outbox/InMemoryOutboxStore.ts](../../src/lib/outbox/InMemoryOutboxStore.ts)

One of these is unused. Risk: confusion about which to import, diverging implementations.

---

### 2. Missing public barrel (`src/index.ts`)
`package.json` declares `"main": "index.js"` but there is **no `src/index.ts`**. Consumers have no clear public API surface. Everything must be imported by deep path.

---

## 🟡 Design Issues

### 3. `IdaeApiDeliverer` return type mismatch
`IdaeApiDeliverer.deliver()` returns `Promise<boolean>` (`DelivererResult = boolean`) while `SyncAdapter` and `OutboxDeliverer` expect `Promise<{ status: 'success'|'retry'|'permanent' }>`. These are **incompatible interfaces** — wiring them together requires an adapter shim or they silently break.

### 4. `createSyncAdapter` in SyncAdapter.ts is a free function wrapping a class
`SyncAdapter.ts` exports both a `class SyncAdapter` and a `function createSyncAdapter` which returns a `new SyncAdapter(...)`. The free function in `initSync.ts` also calls `createSyncAdapter` but passes `(entry) => deliverer.deliver(entry)` as the deliverer, not the typed `Deliverer` interface. API surface is inconsistent.

### 5. `SyncAdapter.ts` contains `IdbqlEventPayload` and `createSyncAdapter` (functional) in the same file as the class
The functional variant of `createSyncAdapter` at the bottom of `SyncAdapter.ts` appears to be a second, independent implementation sharing the same function name. This will cause a **duplicate identifier** compile error or one silently shadows the other.

---

## 🟠 Type Safety

### 6. Pervasive `any` usage (25+ occurrences)
Key locations with unjustified `any`:
| File | Issue |
|------|-------|
| `IdaeApiDeliverer.ts:7,9` | `private client: any` — should type against `IdaeApiClient` |
| `initSync.ts:20,27` | `getIdbqlEvent() as any`, `Array<any>` — event bus type is unknown |
| `SyncAdapter.ts:79,85` | `entry.meta.failed = true as any` — meta type is too narrow |
| `OutboxEntry` fields | `key`, `data`, `whereClause` typed as `any` — acceptable at boundary but undocumented |
| `ConflictResolver.ts` | `local: any, remote: any` — generics would be safer |
| `WhereSerializer.ts` | All `any` — reasonable for a serializer but worth a comment |

### 7. `meta` type inconsistency
`OutboxEntry.meta` is typed as `{ retryCount: number; ... failed?: boolean }` but code does `entry.meta = { ...current, nextAttempt: ... } as any` and `(entry.meta as any).failed = true` — the type doesn't include `nextAttempt`, forcing `as any` casts throughout.

---

## 🟢 Positives

- Good test coverage: 10 test files covering all major modules including e2e
- Exponential backoff in `OutboxDeliverer` is correct (`backoffBaseMs * 2^retryCount`)
- `defaultOnConflict` uses `updated_at` timestamps — sensible LWW strategy
- Error handling wraps all delivery paths; permanent vs transient distinction is clean
- `metrics.ts` is minimal and side-effect free (easy to reset in tests)

---

## Recommended Actions (Priority Order)

1. **Delete** `src/lib/OutboxStore.ts` (legacy) and `src/lib/impl/InMemoryOutboxStore.ts` after verifying no imports remain
2. **Create** `src/index.ts` barrel exporting the public API: `initSync`, `OutboxStore`, `SyncAdapter`, `createSyncAdapter`, `ConflictResolver`, `IdaeApiDeliverer`
3. **Fix** `IdaeApiDeliverer.DelivererResult` — align with `{ status: 'success'|'retry'|'permanent' }` or add an adapter
4. **Fix** duplicate `createSyncAdapter` in `SyncAdapter.ts` — one must be removed
5. **Extend** `OutboxEntry.meta` type to include `nextAttempt?: string` and `failed?: boolean` so `as any` casts are no longer needed
6. **Add** tsconfig.json (missing) — currently compilation settings are implicit/inherited
