## 🔴 Broken / Runtime Risk

### 1. Conflict resolution is a stub
`SyncAdapter.ts:77` accepts `onConflict?: OnConflictHook` param but **never calls it**. No conflict detection runs at runtime. `ConflictResolver.ts` exports functions that are never wired. Declared in spec, not implemented.

### 2. `createSyncAdapter` overloads dead
`SyncAdapter.ts:442-489` — two overload signatures declare `serializer`, `ensureUpdatedAt`, `conflictResolver` params. **Implementation ignores all three.** Dead API surface that misleads callers.

### 3. `ServerFirstHandler` crash risk
`SyncAdapter.ts:342-376` — `applyServerFirst()` uses `this.serverFirstHandler` but it's only initialized when `opts.getDb` is provided. Missing `getDb` → `undefined` method call → crash. No guard.

### 4. `DeliverResult.conflict` never used
`deliverer/IDeliverer.ts:8` — `conflict?: { local, remote }` declared in return type, never set by any deliverer, never read by `SyncAdapter`. Dead field.

## 🟡 Dubious

### 5. Two deliverer classes, unclear canonical
- `Deliverer.ts` → `OutboxDeliverer` (wraps `IDeliverer`, handles retries/backoff)
- `deliverer/IdaeApiDeliverer.ts` → `IdaeApiDeliverer` (HTTP delivery via idae-api)
These serve different layers but the split is implicit. `initSync` wires `IdaeApiDeliverer` via `createIdaeApiDeliverer`, `OutboxDeliverer` wraps it — but this chain is not documented.

### 6. DLQ cast `any` without type guard
`initSync.ts:103-107` — `dlq.listDlq()`, `dlq.replayDlq()`, `dlq.clearDlq()` called via `as any` cast. If `OutboxStore` doesn't implement them, silent failure at runtime. No interface enforcement.

### 7. `InMemoryOutboxStore` test helpers in prod barrel
`outbox/InMemoryOutboxStore.ts` exports `transactionalAppend`, `_applyEntry`, `Tx` — test-only internals visible in public API. From prior audit (2026-03-15): likely dead `src/lib/impl/InMemoryOutboxStore.ts` still not cleaned up.

### 8. `OnConflictHook.context.op` never used
All conflict functions (`defaultOnConflict`, `mergeObjects`, `mergeFieldLevel`) ignore the `op` field. Per-operation conflict logic can't be implemented without it.

---

## 🟢 Minor

### 9. `RollbackManager` snapshots no TTL
`RollbackManager.ts:11` — `Map` of snapshots grows unbounded. Long-running apps → memory leak.

### 10. `SyncModeManager.getCollection()` unnecessary indirection
`SyncModeManager.ts:47-49` — calls `resolve()` instead of direct `collectionModes.get()`. Harmless but noisy.

### 11. `WhereSerializer.serialize` no idempotency check
Already-serialized strings re-encoded. Minor correctness risk if called twice.
