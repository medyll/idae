## Issues Found

### Important

#### I1. Prettier parse errors blocking `pnpm run lint`
**Files:** `bmad/status.yaml`, `bmad/artifacts/stories/S25-04.md`
**Detail:** Two files caused Prettier to crash with `SyntaxError`, masking the normal format-drift warnings.
- `bmad/status.yaml`: folded block scalar (`>`) containing em-dashes (`—`) breaks the YAML parser.
- `bmad/artifacts/stories/S25-04.md`: a ` ```svelte ` code block contains pseudo-code (`import PaneLeft, Frame, DevResetPanel...`) that is invalid Svelte syntax, causing the Markdown parser to throw when it tries to format the embedded code.
**Fix applied during audit:**
- Added `bmad/status.yaml` to `.prettierignore`.
- Changed the language tag in `S25-04.md` from `svelte` to `text` so Prettier skips the invalid block.
**Remaining:** 270 files with format drift (see I3).

#### I2. Circular dependency — `machine.ts` ↔ `MachineAction.ts`
**Files:** `src/lib/main/machine.ts`, `src/lib/main/machine/MachineAction.ts`
**Detail:** `machine.ts` imports `machineActionCallable` from `MachineAction.ts`. `MachineAction.ts` imports the singleton `machine` from `machine.ts`. While ESM lazy evaluation prevents an immediate crash, this creates a tight coupling that makes unit-testing `MachineAction` in isolation impossible and risks load-order fragility.
**Fix:** Inject the `machine` reference into `MachineAction` via constructor or a `setMachine(ref)` setter, rather than importing the singleton directly. `machine.ts` can then instantiate and wire it cleanly.

#### I3. Massive format drift (~170+ files)
**Detail:** `pnpm exec prettier --check .` reports 376 lines of warnings spanning TS, Svelte, JSON, and MD files. The drift has grown since the last audit (2026-05-24: 251 files). This makes PR diffs noisy and erodes style consistency.
**Fix:** Run `pnpm run format` once at root and enforce it in CI.

#### I4. `DataList.svelte` — `state_referenced_locally` warning = reactivity gap
**File:** `src/lib/data-ui/data/DataList.svelte:268`
**Detail:**
```svelte
let visibleCount = $state(pageSize > 0 ? pageSize : 20);
```
This only captures the initial value of the `pageSize` prop. The reset `$effect` watches `collection` and `effectiveWhere` but **not** `pageSize`. If a parent changes `pageSize` dynamically (e.g. a user-selected page-size dropdown), `visibleCount` will not reset, leading to incorrect infinite-scroll chunks.
**Fix:** Add `void pageSize;` to the dependency list in the reset `$effect`:
```svelte
$effect(() => {
    void collection;
    void effectiveWhere;
    void pageSize;
    untrack(() => { visibleCount = chunkSize; });
});
```

### Minor

#### M1. Old `shell/` directory still active — contradicts conventions
**Files:** `src/lib/shell/**/*.svelte`
**Detail:** `conventions.md` states layout components belong in `src/lib/main-ui/layout/`, yet 15+ components (Explorer, TaskBar, Pane, Frame, Navigation, etc.) still live under `src/lib/shell/`. New code (DataList, DataForm, DataFields) lives in `data-ui/` and `main-ui/`, creating a split-brain directory structure.
**Fix:** Schedule a debt story to migrate `shell/` → `main-ui/` (or `data-ui/` where appropriate) and update all imports.

#### M2. i18n leakage in `DataForm.svelte`
**File:** `src/lib/data-ui/data/DataForm.svelte:127`
**Detail:** Error message is hard-coded in French: `"Collection '${collection}' non trouvée dans le schéma."`. The rest of the UI uses English labels (e.g. `'Valider'` is also French, but that's a button label; the error message is more glaring).
**Fix:** Use an i18n key or switch to English for consistency.

#### M3. `MachineAction.ts` — `crypto.randomUUID()` without fallback
**File:** `src/lib/main/machine/MachineAction.ts:72`
**Detail:** Uses `crypto.randomUUID()` directly. In non-secure HTTP contexts this throws. The app likely always runs on localhost/HTTPS, but a defensive fallback is safer.
**Fix:** `payload.id ??= crypto.randomUUID?.() ?? Math.random().toString(36).slice(2);`

#### M4. Mixed import paths in `DataList.svelte`
**File:** `src/lib/data-ui/data/DataList.svelte`
**Detail:** Imports `DataToolbar` from `$lib/shell/layout/DataToolbar.svelte` while other imports come from `data-ui/`. This deepens the coupling to the legacy `shell/` directory.
**Fix:** Move `DataToolbar` to `main-ui/layout/` or `data-ui/controls/` and update the import.
