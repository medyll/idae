### Directory Structure Issues

**Upward Dependency (BL-02 — Confirmed):**
```
data-ui/data/DataList.svelte  →  imports from  shell/frame/explorerUtils.ts
data-ui/data/DataFk.svelte    →  imports from  shell/frame/explorerUtils.ts
data-ui/data/DataRfk.svelte   →  imports from  shell/frame/explorerUtils.ts
data-ui/data/DataFields.svelte → imports from  shell/frame/explorerUtils.ts
```
`data-ui/` (data layer) depends on `shell/` (UI layer) — **inverted dependency**.
`explorerUtils.ts` contains pure functions (`sortItems`, `groupItems`) — should live in `utils/` or `data-ui/utils/`.

**Stale Directory:**
`main-ui/` contains 1 file — likely a leftover from the data-ui/shell refactor (Sprint 23). Should be verified and cleaned.

**Coupling Analysis:**
- `machine.ts` imports from 8 different submodules — orchestrator role, acceptable
- `DataForm.svelte` imports `SchemeFieldDefaultValues` from main/ — acceptable (data → logic)
- All shell/ components import from `$lib/main/machine.js` — expected (UI → core)
- No business logic coupled to infrastructure — clean separation
