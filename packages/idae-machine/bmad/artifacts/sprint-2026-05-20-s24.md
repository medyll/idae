# Sprint 24 — Frame Manager

**Date:** 2026-05-20  
**Goal:** Implémenter `MachineFrameManager`, `machine.loadFrame()`, le composant `<Frame>` Svelte et la `<TaskBar>` — système complet de gestion multi-frames inspiré du pattern `windowGui`/`appGui` legacy et de l'architecture DOM-first de idae-dashboard.

---

## Contexte architectural

### Principes (validés en session)

1. **DOM autoritaire** — le DOM est source de vérité, le registre ne contient que des handles de contrôle
2. **Auto-registration** — chaque `<Frame>` s'enregistre elle-même au mount avec ses propres fonctions
3. **Registry = `Map<frameId, FrameControls>`** — pas d'état, des pointeurs vers les méthodes du composant
4. **`load()` = module dans frame existante** — frame connue → `controls.load()` ; inconnue → `mount(<Frame>)` dans zone DOM → auto-register → `controls.load()`

### Signatures (ordre logique aligné)

```ts
// Zone nommée explicite (frame statique, ex: "main")
machine.loadIn(modulePath, targetId, collection, collectionId?, vars?)

// Frame dynamique — frameId calculé depuis le contenu
machine.loadFrame(modulePath, collection, collectionId?, vars?)
```

### ID déterministique

```ts
computeFrameId('vehicle')              // → "vehicle"
computeFrameId('vehicle', '42')        // → "vehicle:42"
computeFrameId('vehicle', '42', {tab}) // → "vehicle:42:tab=info"
```

### S24-01 — MachineFrameManager singleton + FrameControls interface
**Effort:** S  
**Fichier:** `src/lib/main/frame/MachineFrameManager.ts`

```ts
export interface FrameControls {
  load: (modulePath: string, collection: string, collectionId?: string, vars?: Record<string, string>) => void;
  show: () => void;
  hide: () => void;
  toggle: () => void;
  close: () => void;
}

class MachineFrameManager {
  private registry = new Map<string, FrameControls>();

  register(frameId: string, controls: FrameControls): void
  unregister(frameId: string): void
  load(frameId: string, modulePath: string, collection: string, collectionId?: string, vars?: Record<string, string>): void
    // DOM-first: zone [data-target-zone="${frameId}"] doit exister ou frameId dans registry
    // frame connue → controls.load()
    // inconnue → mount(<Frame>) dans zone DOM → auto-register → controls.load()
  show(frameId: string): void
  hide(frameId: string): void
  toggle(frameId: string): void
  close(frameId: string): void
  get openFrames(): ReadonlyMap<string, FrameControls>
}

export const machineFrameManager = new MachineFrameManager();
```

**AC:**
- Singleton exporté
- `register` / `unregister` fonctionnent
- `load` : frame connue → `controls.load()` ; zone DOM trouvée → `mount(<Frame>)` avant load
- `openFrames` retourne la Map courante (lecture seule)

### S24-02 — computeFrameId() — ID déterministique
**Effort:** S  
**Fichier:** `src/lib/main/frame/frameUtils.ts`

```ts
export function computeFrameId(
  collection: string,
  collectionId?: string,
  vars?: Record<string, string>
): string {
  const base = collectionId ? `${collection}:${collectionId}` : collection;
  if (!vars || Object.keys(vars).length === 0) return base;
  const varStr = Object.keys(vars).sort().map(k => `${k}=${vars[k]}`).join('&');
  return `${base}:${varStr}`;
}
```

**AC:**
- `computeFrameId('vehicle')` → `"vehicle"`
- `computeFrameId('vehicle', '42')` → `"vehicle:42"`
- `computeFrameId('vehicle', '42', { tab: 'info' })` → `"vehicle:42:tab=info"`
- Vars triées → résultat identique peu importe l'ordre des clés

loadFrame(
  modulePath: string,
  collection: string,
  collectionId?: string,

### S24-04 — `<Frame>` Svelte component
**Effort:** M  
**Fichier:** `src/lib/shell/frame/Frame.svelte`

Props:
```ts
{ id: string; modulePath?: string; collection?: string; collectionId?: string; vars?: Record<string, string> }
```

Comportement:
- Mount → `machineFrameManager.register(id, { load, show, hide, toggle, close })`
- Unmount → `machineFrameManager.unregister(id)`
- `load(modulePath, collection, collectionId?)` → `mount(resolvedComponent, { target: bodyEl })` via `componentRegistry.resolve(modulePath)`
- `show()` / `hide()` → toggle CSS `display` (pas unmount, DOM préservé)
- `toggle()` → alterne visible/caché
- `close()` → `unmount()` du module courant + `unregister()`

Structure DOM:
```html
<div class="frame" id="frame-{id}" data-frame-id="{id}">
  <div class="frame-body" id="frame-body-{id}"></div>
</div>
```

**AC:**
- S'auto-enregistre au mount, se désenregistre à l'unmount
- Module monté dans `frame-body-{id}` via componentRegistry
- show/hide = CSS display, pas de remount
- Fonctionne avec modules dynamiques (ExplorerList, CardForm, etc.)
