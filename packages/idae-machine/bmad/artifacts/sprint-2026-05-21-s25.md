# Sprint 25 — Unified Explorer Architecture

**Goal:** AppShell as universal frame template · Explorer single mode-based component · loadFrame as primary navigation API · PaneLeft + 4 Explorer* components deleted

**Date:** 2026-05-21
**Status:** planned

---

## Context

Post-S24 audit revealed architectural fragmentation:
- `ExplorerList`, `ExplorerTable`, `ExplorerCard`, `ExplorerActions` = 4 separate components in componentRegistry — duplicated nav logic, inconsistent props
- `PaneLeft` lives in `layout/` but is semantically a collection navigator (belongs in explorer layer or dies)
- `AppShell.svelte` exists but is unused — all layout is hardcoded inline in `+layout.svelte`
- `loadFrame()` implemented (S24) but never called in production — `loadIn()` used everywhere instead

## Target Architecture

```
AppShell (universal frame template)
  ├── {#snippet navbar}   → app title + dev controls
  ├── {#snippet sidebar}  → DataList[appscheme] — collection nav list
  └── {#snippet content}  → Frame[main] — dynamic mount zone

Navigation flow:
  sidebar click → machine.loadFrame('explorer', collection, undefined, { mode: 'list' })
    → Frame[main] mounts Explorer(collection, mode='list')
    → record click → machine.loadFrame('explorer', collection, recordId, { mode: 'card' })
    → Frame[main] updates Explorer(collection, mode='card', collectionId)

componentRegistry:
  BEFORE: 'explorer.list', 'explorer.table', 'explorer.card', 'explorer.actions'
  AFTER:  'explorer' → Explorer.svelte
```

### S25-02 — Create Explorer.svelte (effort: L)

**What:** Single component replacing ExplorerList + ExplorerTable + ExplorerCard + ExplorerActions.

**Acceptance:**
- Props:
  ```ts
  collection: string
  mode?: 'list' | 'table' | 'card' | 'actions'  // default: 'list'
  collectionId?: string                           // required for mode='card'
  vars?: Record<string, string>
  ```
- Internally uses `DataList collection={collection}` as data layer
- Dispatch per mode:
  - `'list'`    → rows view (was ExplorerList): sortable, paginated, click row → loadFrame card mode
  - `'table'`   → table view (was ExplorerTable): column headers, sort on click
  - `'card'`    → single record form (was ExplorerCard/CardForm): show/edit via mode prop on CardForm
  - `'actions'` → action menu list (was ExplorerActions)
- Record click in list/table → `machine.loadFrame('explorer', collection, record.id, { mode: 'card' })`
- CardForm in card mode: read `collectionId` prop to load the record
- No `machine.loadIn()` — only `machine.loadFrame()` for navigation
- Svelte 5 runes only

**Files:**
- `src/lib/shell/explorer/Explorer.svelte` — CREATE

**Internals (sub-render per mode, inline or extracted to fragments):**
```svelte
{#if mode === 'list'}
  <DataList {collection} ...>
    {#snippet children({items})}
      {#each items as row} ... loadFrame card ... {/each}
    {/snippet}
  </DataList>
{:else if mode === 'table'}
  <DataList {collection} ...>
    ...table render...
  </DataList>
{:else if mode === 'card'}
  <!-- load single record via collectionId -->
  <CardForm {collection} id={collectionId} mode="update" />
{:else if mode === 'actions'}
  <DataList {collection} ...>
    ...action list render...
  </DataList>
{/if}
```
