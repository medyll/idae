## S22-02 — machine.loadIn() + URL builder

**Fichier:** `src/lib/main/machine.ts`

**Description:**  
Méthode `machine.loadIn(modulePath, targetId, collection, collectionId?, vars?)` sur la classe `Machine`.  
Construit l'URL `+targetId/modulePath/collection/collectionId?vars` et appelle `machine.router.push(url)`.  
Multi-appels = segments imbriqués via `+` (cf §11.2).

**Signature:**
```ts
loadIn(modulePath: string, targetId: string, collection: string, collectionId?: string, vars?: string): void
```

**Acceptance criteria:**
- `machine.loadIn('explorer.list', 'main', 'vehicle')` → push `/+main/explorer.list/vehicle`
- `machine.loadIn('card.edit', 'main.modal', 'vehicle', '42', 'tab=info')` → push `/+main.modal/card.edit/vehicle/42?tab=info`
- URL builder extrait en `buildLoadInUrl(...)` pur (testable sans router)

**Tests:** `machineLoadIn.test.ts` — buildLoadInUrl() unitaire, cas sans id, avec vars, avec id.

## S22-03 — URL parser multi-cibles

**Fichier:** `src/lib/main/router/urlParser.ts`

**Description:**  
Parse URL `+targetId` → liste `LoadInSegment[]`. Splitte sur `+`, extrait `{ targetId, modulePath, collection, collectionId?, vars? }`.

**Types:**
```ts
interface LoadInSegment {
  targetId: string;
  modulePath: string;
  collection: string;
  collectionId?: string;
  vars?: Record<string, string>;
}
```

**Acceptance criteria:**
- Parse `/+main/explorer.list/vehicle` → `[{ targetId:'main', modulePath:'explorer.list', collection:'vehicle' }]`
- Parse `/+main/explorer.split/vehicle/42/+main.modal/card.edit/vehicle/42` → 2 segments corrects
- URL sans `+` → throw ou retourne `[]` (à décider dans impl — documenter)
- Querystring du dernier segment parsée dans `vars`

**Tests:** `urlParser.test.ts` — mono-cible, multi-cibles, avec/sans id, avec vars.

## S22-04 — SchemaRouter refactor

**Fichier:** `src/lib/main/router/SchemaRouter.ts`

**Description:**  
Corriger tous les bugs listés en §6 de NAVIGATION.md. Remplacer stubs HTML string par Svelte `mount()`/`unmount()`. Router résout segments via `urlParser` → `componentRegistry` → mount dans `[data-target-zone]`.

**Bugs à corriger (§6):**
- `this.router?.beforeEach` → `.before()` (idae-router API)
- `outlet: '#app'` → `'[data-idae-outlet]'` (ou supprimé — outlet géré par action)
- `base: '/app'` → `'/'`
- `authEnabled: true` → `false` (jusqu'à Phase 4)
- Render stubs → Svelte `mount()`/`unmount()` via componentRegistry

**Route action pattern:**
```ts
action: async (ctx) => {
  const segments = parseLoadInUrl(ctx.path);
  const cleanups: (() => void)[] = [];
  for (const seg of segments) {
    const Comp = await componentRegistry.resolve(seg.modulePath);
    const target = document.querySelector(`[data-target-zone="${seg.targetId}"]`);
    if (!target) continue;
    const app = mount(Comp, { target, props: { collection: seg.collection, dataId: seg.collectionId } });
    cleanups.push(() => unmount(app));
  }
  return () => cleanups.forEach(fn => fn());
}
```

**Acceptance criteria:**
- `beforeEach` remplacé par `before`
- `authEnabled: false` par défaut
- `base: '/'`, outlet géré par action (pas option globale)
- Toute action utilise `mount()`/`unmount()` — zéro HTML string
- Pas de `goto()` dans SchemaRouter

**Tests:** Vitest — guard appelle `before()`, action monte composant dans target, cleanup appelé sur leave.
