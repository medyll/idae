# Story S6-01 — Fix moduleDbName() + Decouple MongoDB URI

**Sprint:** 6  
**Goal:** Fix DB naming convention + remove hardcoded DB name from MongoDB URI

## Problem

Two bugs block multi-DB routing:

1. `machine.moduleDbName(module)` builds `{org}_{domain}_{module}` — wrong.  
   Arch decision: `{org}_{base}` where `base` comes from `CollectionModel.base`.

2. `config.mongodbUri` defaults to `'mongodb://localhost:27017/idae_machine'` —  
   the hardcoded DB name prevents per-request `useDb()` routing.

## Changes

| File | Change |
|------|--------|
| `src/lib/main/machine.ts` | `moduleDbName(base)` → `${org}_${base}` |
| `server/src/config.ts` | default URI → `'mongodb://localhost:27017'` |

## Acceptance Criteria

- [ ] `machine.moduleDbName('machine_base')` with `org='test'` returns `'test_machine_base'`
- [ ] `machine.moduleDbName('machine_app')` with `org='test'` returns `'test_machine_app'`
- [ ] Default `config.mongodbUri` has no database name suffix
- [ ] `pnpm run check` passes in both root and server/
