## 2. MachineScheme.defaultSort getter

Inference rules — **ordered priority**:

| Priority | Condition | Result |
|----------|-----------|--------|
| 1 | `template.sort` is set | return `template.sort` as-is |
| 2 | field named `ordre` or `order` exists | `[{ field: 'ordre'/'order', direction: 'asc' }]` |
| 3 | field named `name`, `nom`, `label`, `titre`, `title`, `code` exists (first match) | `[{ field, direction: 'asc' }]` |
| 4 | field named `updatedAt` or `dateModification` exists | `[{ field, direction: 'desc' }]` |
| 5 | field named `createdAt` or `dateCreation` exists | `[{ field, direction: 'desc' }]` |
| 6 | any field with type `date` or `datetime` (first found) | `[{ field, direction: 'desc' }]` |
| 7 | fallback | `[{ field: this.index, direction: 'asc' }]` |

```ts
get defaultSort(): SortBy[] {
  const tplSort = this.template?.sort;
  if (tplSort?.length) return tplSort;

  const fields = this.fields;
  const fieldNames = Object.keys(fields);

  // Priority 2 — manual ordre
  const ordreField = fieldNames.find(f => f === 'ordre' || f === 'order');
  if (ordreField) return [{ field: ordreField, direction: 'asc' }];

  // Priority 3 — name/label family (legacy: nom + code)
  const nameField = fieldNames.find(f =>
    ['name', 'nom', 'label', 'titre', 'title', 'code'].includes(f)
  );
  if (nameField) return [{ field: nameField, direction: 'asc' }];

  // Priority 4/5 — timestamps
  const updatedField = fieldNames.find(f => f === 'updatedAt' || f === 'dateModification');
  if (updatedField) return [{ field: updatedField, direction: 'desc' }];
  const createdField = fieldNames.find(f => f === 'createdAt' || f === 'dateCreation');
  if (createdField) return [{ field: createdField, direction: 'desc' }];

  // Priority 6 — any date type
  const dateField = fieldNames.find(f => {
    const t = fields[f]?.type ?? '';
    return t === 'date' || t === 'datetime';
  });
  if (dateField) return [{ field: dateField, direction: 'desc' }];

  // Fallback
  return [{ field: this.index, direction: 'asc' }];
}
```

If `sort` omitted → `defaultSort` getter infers from fields. For `vehicles` with `name` + `createdAt`:
→ inferred: `[{ field: 'name', direction: 'asc' }]` (priority 3 wins over priority 5).

## Acceptance criteria

1. Collection with `template.sort` set → `defaultSort` returns exactly that array
2. Collection with field `ordre` → `[{ field: 'ordre', direction: 'asc' }]`
3. Collection with `code` + `createdAt` → `[{ field: 'code', direction: 'asc' }]` (priority 3 > 5)
4. Collection with `nom` + `dateModification` → `[{ field: 'nom', direction: 'asc' }]`
5. Collection with only `updatedAt` (no name/label/code) → `[{ field: 'updatedAt', direction: 'desc' }]`
6. Collection with only a `date`-type field → `[{ field, direction: 'desc' }]`
7. Collection with no matching field → `[{ field: keyPath-index, direction: 'asc' }]`
8. `ExplorerList` with no `sortBy` prop → reads `defaultSort` from scheme → items sorted automatically
9. `ExplorerList` with explicit `sortBy` prop → `sortBy` wins, `defaultSort` ignored
10. Multiple sort chain: `[{ field: 'status', direction: 'asc' }, { field: 'name', direction: 'asc' }]` → items sorted by status first, then name as tiebreaker
11. `sortItems` single `SortBy` still works (backward compat)
