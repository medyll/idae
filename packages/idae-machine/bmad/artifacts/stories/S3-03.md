# Story S3-03: Configure conflict resolution strategies

**Sprint:** S3 — Real-Time  
**Status:** Ready for Development  
**Priority:** High  
**Estimated Effort:** 1-2 days

---

## User Story

**As a** developer  
**I want** to configure conflict resolution strategies  
**So that** I can handle concurrent edits appropriately

---

## Acceptance Criteria

- [ ] `last-write-wins` strategy (default)
- [ ] `server-wins` strategy
- [ ] `client-wins` strategy
- [ ] `manual` strategy (returns conflict to client)
- [ ] Strategy configurable per table
- [ ] Conflict event emitted to client
- [ ] Audit trail for conflicts

---

## Technical Requirements

### Strategy Types

```typescript
type ConflictStrategy = 
  | 'last-write-wins'
  | 'server-wins'
  | 'client-wins'
  | 'manual';
```

### Configuration

```typescript
const syncConfig = {
  defaultStrategy: 'last-write-wins',
  tableStrategies: {
    'financial_records': 'server-wins',
    'comments': 'client-wins',
    'documents': 'manual'
  }
};
```

### Conflict Event

```typescript
interface Conflict {
  table: string;
  id: string;
  clientVersion: Record;
  serverVersion: Record;
  strategy: ConflictStrategy;
  resolved?: Record;
}
```
