## User Story

As a **form developer**, I want to **support custom validators, cross-field validation, and async validation**, so that **complex business rules (email uniqueness, password strength, date ranges) can be enforced at the form level**.

---

## Context

Baseline audit found that `MachineSchemeValidate.ts` only handles basic required/readonly/type checking. Missing:
- Custom validator functions per field
- Cross-field validation (e.g., endDate > startDate)
- Async validators (debounced API calls)
- Comprehensive error messaging

**Linked Audit Finding**: "Form Validation Incomplete"  
**Linked PRD Requirements**: FR-3.5 through FR-3.9  
**Linked Tech Spec Section**: API Design → Field Validation API

### Features Implemented

**1. ValidationError Interface**
```typescript
export interface ValidationError {
  fieldName: string;
  message: string;
  severity: 'error' | 'warning';
  type: 'required' | 'type' | 'format' | 'custom' | 'cross-field';
}
```

**2. Custom Validator Registration**
- `registerCustom(fieldName, validator)` — Register per-field validation functions
- Validators are stored in Map<fieldName, fn[]>
- Called during validateField() after type validation
- Returns error if any custom validator returns false

**3. Async Validator Support**
- `registerAsync(fieldName, validator)` — Register async validators (Promise<boolean>)
- Useful for API-based checks (email uniqueness, username availability, etc.)
- Called sequentially after custom validators
- Properly awaited with error handling

**4. Cross-Field Validation**
- `registerCrossField({ fields, validator })` — Register dependent field validation
- Examples: date range validation, password matching, conditional requirements
- Called in validateForm() after all field validations
- Supports both sync and async validators

### Internal Registries
- `customValidators: Map<string, Array<(value: unknown) => boolean>>`
- `asyncValidators: Map<string, Array<(value: unknown) => Promise<boolean>>>`
- `crossFieldValidators: Array<{ fields: string[]; validator }>`

### Testing
- All 88 tests passing (zero regressions)
- Validators integrated with existing field and form validation
- Error handling preserves existing patterns

### API Usage Examples
```typescript
const validator = schema.validator;

// Custom validator
validator.registerCustom('email', (val) =>
  /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(String(val))
);

// Async validator
validator.registerAsync('username', async (val) => {
  const response = await fetch(`/api/check-username?name=${val}`);
  const data = await response.json();
  return data.available;
});

// Cross-field validator
validator.registerCrossField({
  fields: ['password', 'passwordConfirm'],
  validator: (data) => data.password === data.passwordConfirm
});

// Cross-field date validation
validator.registerCrossField({
  fields: ['startDate', 'endDate'],
  validator: (data) => new Date(data.endDate) > new Date(data.startDate)
});

// Validate form with all validators
const result = await validator.validateForm(formData);
// result.isValid, result.errors, result.invalidFields
```

## Technical Notes

### API Design (from PRD FR-3.5 onwards)

```typescript
interface ValidationError {
  fieldName: string;
  message: string;
  severity: 'error' | 'warning';
  type: 'required' | 'type' | 'format' | 'custom' | 'cross-field';
}

interface MachineSchemeValidate {
  // Existing
  validateField(fieldName: string, value: unknown): void // throws MachineErrorValidation

  // New (S1-02)
  validateForm(formData: Record<string, unknown>): ValidationError[]
  registerCustom(fieldName: string, validator: (value: unknown) => boolean): void
  registerAsync(fieldName: string, validator: (value: unknown) => Promise<boolean>): void
  registerCrossField(rule: {
    fields: string[];
    validator: (data: Record<string, unknown>) => boolean;
  }): void
}
```

### Implementation Strategy

1. **Extend `MachineSchemeValidate` class** (80 lines → 150+ lines)
   - Add custom validator registry (Map<fieldName, fn[]>)
   - Add cross-field validator registry
   - Implement `validateForm()` that runs all validators

2. **Error collection** (not throwing on first error)
   - Collect ALL validation errors per form
   - Return structured `ValidationError[]`

3. **Async validation** (debounced)
   - Use `setTimeout` with 300ms debounce
   - Emit intermediate "checking..." state
   - Handle API failures gracefully (show warning, allow override)

4. **UI integration** (in form components)
   - Display inline errors from `ValidationError[]`
   - Show loading state during async validation
   - Disable submit until all validations pass

### Out of Scope

- Nested object validation (deferred to Sprint-2+)
- Async validator cancellation (use AbortController later)
- Validator chaining / composition (keep simple for v1.0)
