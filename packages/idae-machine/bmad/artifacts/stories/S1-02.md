# Story S1-02: Build schema endpoints

**Sprint:** S1 — Foundation  
**Status:** In Progress  
**Priority:** High  
**Estimated Effort:** 2-3 days

---

## User Story

**As a** developer  
**I want** schema endpoints that serve collection definitions  
**So that** the client can discover data models and generate UI dynamically

---

## Acceptance Criteria

- [ ] `GET /api/scheme` returns array of all schemes
- [ ] `GET /api/scheme/:table` returns single scheme by code
- [ ] Response includes `_views` registry (entityModel, listView, miniView, etc.)
- [ ] 404 returned for unknown table
- [ ] Legacy schemes auto-migrated to _views format
- [ ] Response time < 50ms for cached schemes

---

## Technical Requirements

### API Endpoints

```
GET /api/scheme         → { schemes: AppScheme[] }
GET /api/scheme/:table  → AppScheme
```

### Response Format

```json
{
  "idappscheme": "1",
  "code": "produit",
  "name": "Produits",
  "_views": {
    "entityModel": [...],
    "listView": [...],
    "miniView": [...]
  },
  "fields": [...]
}
```

### Auto-Migration Logic

```typescript
function migrateLegacyScheme(legacy: LegacyAppScheme): AppScheme {
  return {
    ...legacy,
    _views: {
      entityModel: legacy.fieldModel || [],
      listView: legacy.columnModel || [],
      miniView: legacy.miniModel || [],
      fkLabelView: legacy.hasModel || []
    }
  };
}
```
