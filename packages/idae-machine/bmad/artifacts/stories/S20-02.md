## Context

Legacy stockait les fichiers via GridFS dans MongoDB (`LEGACY-ACTIONS-MIGRATION.md` section 6, ligne `GridFS documents`). idae-machine n'a actuellement aucun service fichiers.

Goal : service unifié upload/download/delete avec storage **disque local** par défaut, structure prévisible, métadonnées en MongoDB, multi-tenant (par `org`).

Layout disque :
```
server/data/files/{org}/{collection}/{recordId}/{fileId}-{originalName}
```

Collection metadata MongoDB : `appfile` (par base `{org}_machine_user` — partagée avec audit).

```ts
{
  _id:         ObjectId,
  fileId:      string,         // uuid v4
  collection:  string,         // ex: 'invoices'
  recordId:    string,         // ex: '507f1f77...'
  originalName: string,
  storedPath:  string,         // relatif à server/data/files/{org}/
  size:        number,
  mimetype:    string,
  uploadedAt:  string,         // ISO
  uploadedBy?: string,         // userId
  deletedAt?:  string | null,
}
```

## API service

```ts
// server/src/services/FileService.ts
export interface UploadFileInput {
	org:          string;
	collection:   string;
	recordId:     string;
	originalName: string;
	mimetype:     string;
	size:         number;
	stream:       NodeJS.ReadableStream;    // multer file stream
	uploadedBy?:  string;
}

export interface FileMeta {
	fileId:       string;
	collection:   string;
	recordId:     string;
	originalName: string;
	size:         number;
	mimetype:     string;
	uploadedAt:   string;
	uploadedBy?:  string;
}

export async function uploadFile(input: UploadFileInput): Promise<FileMeta>;
export async function getFileMeta(org: string, fileId: string): Promise<FileMeta | null>;
export async function streamFile(org: string, fileId: string): Promise<{ stream: NodeJS.ReadableStream; meta: FileMeta } | null>;
export async function deleteFile(org: string, fileId: string, permanent?: boolean): Promise<boolean>;
export async function listFiles(org: string, collection: string, recordId: string): Promise<FileMeta[]>;
```

Soft delete par défaut (champ `deletedAt`) — cohérent avec `data.ts`. `permanent: true` supprime le fichier disque + metadata.

## Routes REST

| Method | Path | Permission | Handler |
|--------|------|------------|---------|
| POST   | `/api/files/:collection/:recordId` (multipart) | `requireDroit('U')` sur collection | `uploadFile` |
| GET    | `/api/files/:collection/:recordId` | `requireDroit('R')` | `listFiles` |
| GET    | `/api/files/:fileId/meta` | `requireDroit('R')` | `getFileMeta` |
| GET    | `/api/files/:fileId/download` | `requireDroit('R')` | `streamFile` (Content-Disposition: attachment) |
| DELETE | `/api/files/:fileId` | `requireDroit('D')` | `deleteFile` (soft, `?permanent=true` admin) |

`requireDroit` actuel filtre par `:table` param. Pour `/api/files/:fileId/...`, charger meta d'abord pour récupérer `collection`, puis check droit. Helper middleware `requireFileAccess(perm)`.

## Multer config

```ts
// server/src/middleware/upload.ts
import multer from 'multer';
import path from 'path';
import { randomUUID } from 'crypto';
import fs from 'fs';
import { config } from '../config.js';

const storage = multer.diskStorage({
	destination(req, _file, cb) {
		const { collection, recordId } = req.params;
		const org = (req as any).org ?? 'default';
		const dir = path.join(config.filesRoot, org, collection, recordId);
		fs.mkdirSync(dir, { recursive: true });
		cb(null, dir);
	},
	filename(_req, file, cb) {
		const fileId = randomUUID();
		(file as any).fileId = fileId;
		cb(null, `${fileId}-${file.originalname}`);
	},
});

export const uploadMiddleware = multer({
	storage,
	limits: { fileSize: config.maxFileSize },
	fileFilter(_req, file, cb) {
		if (!config.allowedMimeTypes) return cb(null, true);
		cb(null, config.allowedMimeTypes.includes(file.mimetype));
	},
});
```

## Security

- **Path traversal** : `validateTableName(collection)` (déjà dans `data.ts`) + valider `recordId` format ObjectId/uuid. Refus si autre chose.
- **MIME spoofing** : trust `mimetype` du client = NO. Optionnel : `file-type` package pour détection magic-bytes. Hors scope ce sprint, mais doc dans story.
- **Size limit** : `multer.limits.fileSize` + check côté handler avant insert metadata.
- **Filename** : déjà préfixé `fileId-` → pas de collision. Bien laisser `originalName` côté metadata mais utiliser `storedPath` pour servir.

## Acceptance criteria

1. POST multipart → fichier écrit dans `data/files/{org}/{coll}/{id}/{fileId}-{name}`, metadata créée
2. GET `/api/files/:coll/:id` → liste les FileMeta de ce record
3. GET `/api/files/:fileId/download` → stream le fichier avec bon `Content-Type` + `Content-Disposition: attachment; filename="..."`
4. DELETE soft → `deletedAt` set, fichier reste sur disque, plus listé
5. DELETE `?permanent=true` → fichier `unlink` + metadata `deleteOne`
6. Upload `> maxFileSize` → 413 Payload Too Large
7. MIME hors `allowedMimeTypes` (si config) → 415 Unsupported Media Type
8. Path traversal tenté (`../../etc/passwd`) → 400 Invalid table name
9. Permission denied sur collection (RBAC) → 403
10. Suite tests existants no regression
