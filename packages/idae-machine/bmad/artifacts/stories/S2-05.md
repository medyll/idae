### Phase 4: Create Usage Examples (Day 2)

**File: `docs/EXAMPLES.md`**

1. **Form Creation**

   ```typescript
   const validator = new MachineSchemeValidate('agents', machineDb);
   const result = await validator.validateForm(formData);
   ```

2. **Custom Validators**

   ```typescript
   validator.registerCustom('email', (val) => /^.+@.+$/.test(val));
   ```

3. **Async Validators**

   ```typescript
   validator.registerAsync('username', async (val) => {
   	const response = await fetch(`/api/check?name=${val}`);
   	return (await response.json()).available;
   });
   ```

4. **Cross-Field Validators**

   ```typescript
   validator.registerCrossField({
   	fields:    ['password', 'passwordConfirm'],
   	validator: (data) => data.password === data.passwordConfirm
   });
   ```

5. **Complete Form Workflow**
   - Render form
   - Register validators
   - Handle submit
   - Display errors

### Phase 5: Create Migration Guide (Day 2-3)

**File: `docs/MIGRATION.md`**

**From CrudService to MachineSchemeValidate**

- CrudService removed in Sprint-1
- New validation API is more flexible
- Migration path for each use case
- Code examples before/after

**API Changes**

- Old: validateField() threw on error
- New: validateField() returns { isValid, error }
- Old: No custom validators
- New: registerCustom(), registerAsync(), registerCrossField()
