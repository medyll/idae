## Context

Les noms des stores IDB changent (`_prefs` → `appuser_prefs` etc.). IndexedDB requiert un version bump pour modifier les object stores. Sans bump, l'ancienne DB reste et les nouveaux noms ne sont pas créés.

## Files

| File | Change |
|------|--------|
| `src/routes/+layout.svelte` | `version: 1` → `version: 2` |
| `bmad/conventions.md` | Note: IDB version = incrémenter à chaque rename de store |

## Acceptance

- `machine.init({ ..., version: 2 })` dans +layout.svelte
- DevResetPanel "Clear IDB" permet de reset manuellement si besoin
- Pas de migration automatique (contexte dev, données demo)
- Note dans conventions.md: "Chaque rename de collection IDB = bump version dans machine.init()"
