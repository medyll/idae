# S52-01 — BL-19: Right recent-history panel (appuser_history 30d/15, pref app_panel)

## Story

Build the right-zone recent-history panel, legacy `app_gui_panel_list.php` / `app_gui_panel.php`. It lists the user's recent activity per permitted+preferred collection, grouped by collection, capped at 30 days / 15 records.

## Files

- New: `src/lib/shell/layout/RecentPanel.svelte`
- Modified: `src/lib/shell/layout/TemplateShell.svelte` (wire into right zone)
- Export: `src/lib/index.ts`

## Acceptance criteria

1. `RecentPanel.svelte` renders `appuser_history` records filtered to the last 30 days, max 15 per collection.
2. Records are grouped by collection name/type.
3. Visibility follows the menu-pref policy: `filterMenuCollections('panel', ...)` from `menuPrefs.ts`.
4. Dev mode (`import.meta.env.DEV`) bypasses pref filtering.
5. Empty prefs show all permitted collections (BL-12 default policy).
6. Clicking a record navigates to it via framer (e.g. `loadFrame('explorer', collection, id)` or `loadInDialog('form', collection, id)`).
7. No `machine.collection()` reads — use `machine.store('appuser_history', { ... })` (invariant 8).
8. Custom tags have explicit `display` CSS.
9. Component test verifies filtering, grouping, and navigation wiring.
10. `pnpm run check` 0 errors, `pnpm run test` green.
