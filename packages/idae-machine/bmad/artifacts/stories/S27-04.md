## Goal

InputSelect construit les labels d'options FK depuis `template.presentation`. Utiliser `_views.fk_label` à la place si disponible.

## File

| File | Change |
|------|--------|
| `src/lib/data-ui/input/InputSelect.svelte` | Préférer `_views.fk_label` pour label fields |

## Logic

```ts
const fkScheme = $derived(fkCollection ? machine.logic.collection(fkCollection) : null);
const labelFields = $derived(
  fkScheme?.views?.fk_label?.map(f => f.name)
  ?? fkScheme?.template?.presentation?.split(' ')
  ?? ['id']
);
```

Utiliser `labelFields` pour construire le label de chaque option dans le select.

## Acceptance

- Si `_views.fk_label` présent → labels FK utilisent ces champs
- Fallback `template.presentation` si absent — 0 régression comportementale
- Tests InputSelect existants passent
