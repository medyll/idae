# Story S2-02: Build permission middleware (requireDroit)

**Sprint:** S2 — Data Layer  
**Status:** Ready for Development  
**Priority:** High  
**Estimated Effort:** 2-3 days

---

## User Story

**As a** system administrator  
**I want** permission-based access control on all CRUD operations  
**So that** users can only access data they are authorized to see

---

## Acceptance Criteria

- [ ] `requireDroit(permission)` middleware factory
- [ ] Support for C, R, U, D, L, X, A permissions
- [ ] Check permissions before CRUD operations
- [ ] Return 403 Forbidden on permission denied
- [ ] Support for table-level permissions
- [ ] Client-side permission cache
- [ ] Permission check endpoint: `GET /api/permissions/check`

---

## Technical Requirements

### Middleware

```typescript
function requireDroit(permission: Permission, table?: string) {
  return (req: Request, res: Response, next: NextFunction) => {
    // Check user has permission
    // Return 403 if denied
  };
}
```

### Permission Types

```typescript
type Permission = 'C' | 'R' | 'U' | 'D' | 'L' | 'X' | 'A';
// C = Create, R = Read, U = Update, D = Delete
// L = List, X = Execute, A = Admin
```

### Integration

```typescript
// Apply to routes
idaeApi.router.addRoute({
  method: 'get',
  path: '/api/data/:table',
  handler: listRecords,
  middleware: [requireDroit('L')]
});
```
