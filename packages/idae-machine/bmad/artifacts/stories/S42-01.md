# S42-01 — DataList mode unification : supprimer TableInline, svelte:element + DataField

> Sprint 42 | Effort M | Source BL-07 | ADR-06
> Status: ready

---

## Contexte

DataList expose 3 modes via prop `mode?: 'list' | 'table' | 'grid'` (défaut `'list'`).
Le mode courant est `currentMode = $derived(userMode ?? modeProp)` (ligne ~127).

| Mode | Rendu actuel | Status |
|---|---|---|
| `list` | `<ul><li>` + `renderPresentation` / `DataRecord` | ✅ scheme-aware |
| `grid` | `<ul class="grid-list"><li>` + `DataRecord mode="show"` | ✅ scheme-aware |
| `table` | délègue à `<TableInline {collection} {where} {onItemClick}>` (ligne 361) | ❌ scheme-aveugle |

**TableInline défauts :**
- Colonnes = `Object.keys(record).filter(k => k !== 'id' && k !== '_id').slice(0, 8)` — Mongo brut, 8 max
- Valeurs = `{item[col] ?? ''}` — FK non résolues, formats non appliqués, private fields visibles
- Tri = state local `sortColumn`/`sortDirection` — isolé de `userSortBy` de DataList
- Aucun lien avec `machine.logic`, `getFieldsForView`, `DataField`

## Changements à apporter

### 1. Nouveau dérivé `tableColumns` (ajouter après `fullFields`)

```ts
// Colonnes du mode table — depuis scheme, pas Object.keys
// Défaut view='flat' pour exclure FK-only (cohérent avec vue tabulaire)
const tableColumns = $derived.by(() => {
    if (currentMode !== 'table' || !collLogic) return [] as { name: string; label: string }[];
    const viewKey = (view as 'full' | 'flat' | 'fk' | 'focus') ?? 'flat';
    return collLogic.getFieldsForView(viewKey).map(f => ({
        name:  f.name,
        label: String(f.name),   // ← header simple : nom du champ (DataField gère le label FK en cellule)
    }));
});
```

> **Anti-dérive :** Ne pas essayer de résoudre `fkTargetName` dans le header. DataField s'en charge
> en cellule via son propre `fkTargetName`. Le header affiche le nom de champ brut (`category`,
> `location_office`...) — c'est intentionnel, cohérent avec ce que fait `DataRecord` pour ses labels.

### 2. Remplacer le branchement `mode='table'` (ligne ~360-361)

**Avant :**
```svelte
{:else if currentMode === 'table'}
    <TableInline {collection} {where} onItemClick={handleItemClick} />
```

**Après :**
```svelte
{:else if currentMode === 'table'}
    <table class="data-table">
        <thead>
            <tr>
                {#each tableColumns as col (col.name)}
                    <th
                        class:sorted={userSortBy.some(s => s.field === col.name)}
                        onclick={() => {
                            const existing = userSortBy.find(s => s.field === col.name);
                            if (existing) {
                                prefs.set('sortBy', [{ field: col.name, direction: existing.direction === 'asc' ? 'desc' : 'asc' }]);
                            } else {
                                prefs.set('sortBy', [{ field: col.name, direction: 'asc' }]);
                            }
                        }}
                    >
                        {col.label}
                        <span class="sort-arrow">
                            {#if userSortBy.some(s => s.field === col.name)}
                                {userSortBy.find(s => s.field === col.name)?.direction === 'asc' ? '↑' : '↓'}
                            {:else}↕{/if}
                        </span>
                    </th>
                {/each}
            </tr>
        </thead>
        <tbody>
            {#each paginatedItems as record ((record as Record<string, unknown>)[indexField])}
                <tr
                    class:clickable={!!onItemClick || !!parsedLink}
                    onclick={() => handleItemClick(record)}
                >
                    {#each tableColumns as col (col.name)}
                        <td>
                            <DataField
                                {collection}
                                fieldName={col.name}
                                mode="show"
                                data={record as Record<string, unknown>}
                                showLabel={false}
                            />
                        </td>
                    {/each}
                </tr>
            {/each}
            {#if !paginatedItems.length}
                <tr><td colspan={tableColumns.length}>
                    {#if emptySnippet}{@render emptySnippet()}{:else}—{/if}
                </td></tr>
            {/if}
        </tbody>
    </table>
```

> **Anti-dérive :** Le tri passe par `prefs.set('sortBy', ...)` — même chemin que les boutons
> `DataSort` du toolbar. PAS de state local `sortColumn`/`sortDirection`.
> `paginatedItems` est déjà trié via `effectiveSort → sortedItems → paginatedItems`.
> Pas besoin de retrier en table.

## Ce qu'il NE FAUT PAS faire (anti-dérive explicite)

| ❌ Mauvais | ✅ Correct |
|---|---|
| Recréer un composant `DataTable.svelte` séparé | Tout inline dans DataList |
| State local `sortColumn`/`sortDirection` dans DataList | `prefs.set('sortBy', ...)` uniquement |
| Colonnes depuis `Object.keys(record)` | `collLogic.getFieldsForView(viewKey)` |
| Retrier `paginatedItems` dans la table | `paginatedItems` est déjà trié |
| Mettre les styles dans `<style>` scoped DataList | `src/styles/components.css` @layer components |
| Résoudre `fkTargetName` pour les headers | DataField le fait en cellule, header = `col.name` |
| Utiliser `<svelte:element>` pour `<table>` ou `<thead>` | `<svelte:element>` pour `<tbody>` et `<tr>` seulement si nécessaire — structure `<table>` est statique |

> **Note sur svelte:element :** Le branchement list/grid/table a des structures container trop différentes
> (`<ul>` vs `<table>`) pour être unifiées avec `<svelte:element>`. L'intérêt de `<svelte:element>` se
> matérialise davantage à l'intérieur du loop (item = `li` vs `tr`). Si le code s'avère plus clair avec
> 3 branches `{#if}` distinctes, c'est acceptable — l'important est le pipeline DataField unifié, pas
> la réduction syntaxique des tags.
