# Story S1-03: Create MachineApi client class

**Sprint:** S1 — Foundation  
**Status:** In Progress  
**Priority:** High  
**Estimated Effort:** 2 days

---

## User Story

**As a** developer  
**I want** a MachineApi client class  
**So that** I can fetch schemas and data from the server in the frontend

---

## Acceptance Criteria

- [ ] `MachineApi` class with configurable baseUrl
- [ ] `fetchAllSchemes()` returns array of schemes
- [ ] `fetchScheme(table)` returns single scheme
- [ ] `health()` checks server status
- [ ] Error handling with retries (3 attempts)
- [ ] TypeScript types for all methods
- [ ] Response caching with TTL

---

## Technical Requirements

### Interface

```typescript
class MachineApi {
  constructor(options: {
    baseUrl: string;
    timeout?: number;
    retries?: number;
    cacheTtl?: number;
  });

  // Schemas
  fetchAllSchemes(): Promise<AppScheme[]>;
  fetchScheme(table: string): Promise<AppScheme>;
  
  // Health
  health(): Promise<{ status: string; version: string }>;
}
```

### Error Handling

```typescript
class MachineApiError extends Error {
  constructor(
    message: string,
    public statusCode: number,
    public response?: Response
  ) {
    super(message);
  }
}
```
