## User Story

As a **contributor**, I want to **improve JSDoc coverage from 4.5% to 80%+**, so that **IDE autocomplete, parameter hints, and API documentation are clear and complete for all public methods**.

---

## Context

Baseline audit found only 4.5% JSDoc coverage (141 blocks in 3,111 TS lines). Missing:

- `@param` descriptions for most function parameters
- `@return` type documentation
- `@throws` error documentation
- `@role` documentation for major classes
- Component `@slot` documentation (Svelte)

**Linked PRD**: NFR: ">80% JSDoc coverage"  
**Linked Tech Spec**: Type Safety → JSDoc documentation requirement

---

## Acceptance Criteria

```gherkin
Feature: Class-Level Documentation
  Scenario: Machine class documented
    Given Machine.ts (main entry point)
    When reviewing JSDoc
    Then has @class, @role, and overview for all public methods

  Scenario: MachineDb class documented
    Given MachineDb.ts (schema introspection)
    When reviewing JSDoc
    Then has @param, @return, @throws for all public methods

Feature: Method-Level Documentation
  Scenario: Parameter documentation
    Given method signature: validateField(fieldName: string, value: any)
    When reviewing JSDoc
    Then has @param {string} fieldName - description
    And has @param value - description

  Scenario: Return type documentation
    Given method returns an object
    When reviewing JSDoc
    Then has @return {Object} with shape description
    And includes type details (interface or inline)

  Scenario: Error documentation
    Given method throws custom error
    When reviewing JSDoc
    Then has @throws {MachineErrorValidation} when validation fails

Feature: Component Documentation
  Scenario: Component props documented
    Given Svelte component with $props()
    When reviewing JSDoc/inline types
    Then all props have descriptions
    And optional props marked as optional

Feature: Coverage Measurement
  Scenario: Measure JSDoc coverage
    Given all files in src/lib/main/ and src/lib/form/
    When running JSDoc coverage tool
    Then coverage ≥80% (at least 80% of symbols documented)
```

## Component JSDoc Format (Svelte)

For Svelte components, document in:

1. `<script>` top block (component overview)
2. Props interface JSDoc
3. Slot documentation

````svelte
<script lang="ts">
  /**
   * @component CreateUpdate Form Component
   * @role User form for creating or editing records in a collection
   * @description Renders an editable form with automatic field generation from schema.
   * @example
   * ```svelte
   * <CreateUpdate collection="users" mode="create" onSubmit={handleSave} />
   * ```
   */

  interface Props {
    /** Collection name to bind form to (e.g., 'users') */
    collection: string;
    /** Form mode: 'create' (new record), 'edit' (existing), 'show' (readonly) */
    mode: 'create' | 'edit' | 'show';
    /** Initial data for edit/show modes */
    data?: Record<string, any>;
    /** Callback when form is submitted successfully */
    onSubmit?: (record: Record<string, any>) => void;
  }

  let { collection, mode = 'create', data, onSubmit } = $props();
</script>

<!-- @slot default Render custom form field template -->
<slot />
````
