# S27-01 — getModel() query appscheme_view → build _views par collection

**Sprint:** 27
**Effort:** M
**Priority:** high
**Dependencies:** S25 complet, S26 complet

---

## Context

`appscheme_view` est peuplé par `deployModel()` avec les champs de chaque vue (list/mini/form/fk_label) par collection. Mais `MachineServer.getModel()` ignore ces données — `_views` n'est jamais inclus dans le MachineModel retourné au client.

## File

| File | Change |
|------|--------|
| `server/src/MachineServer.ts` | `getModel()` — query appscheme_view, construire `_views` |

## Implementation

Après avoir chargé les `appscheme` records, pour chaque collection:

```ts
// Query appscheme_view pour cette collection
const viewRows = await AppschemeViewModel.find({
  'gridFks.appscheme.code': collectionCode
}).populate('gridFks.appscheme_view_type').populate('gridFks.appscheme_field');

// Grouper par view_type.code
const _views: Partial<EntityViews> = {};
for (const row of viewRows) {
  const viewType = row.gridFks?.appscheme_view_type?.code; // 'list'|'mini'|'form'|'fk_label'
  const fieldCode = row.gridFks?.appscheme_field?.code;
  if (!viewType || !fieldCode) continue;
  if (!_views[viewType]) _views[viewType] = [];
  _views[viewType].push({
    name:  fieldCode,
    code:  fieldCode,
    title: fieldCode.replace(/_/g, ' '),
    order: row.order ?? 0,
  });
}

// Trier par order dans chaque vue
for (const key of Object.keys(_views)) {
  _views[key].sort((a, b) => (a.order ?? 0) - (b.order ?? 0));
}

// Ajouter au model entry
model[code] = { keyPath, base, model: {}, fields, fks, template, _views };
```

## Acceptance

- `_views` présent sur chaque collection dans le MachineModel retourné
- `_views: {}` (objet vide) si aucun appscheme_view pour cette collection (gracieux)
- `_views.list`, `_views.form`, `_views.mini`, `_views.fk_label` triés par `order`
- Pas de crash si appscheme_view collection vide (première install sans deploy)
