## Context

IDB is a cache of the MongoDB schema. When the server schema changes (admin renames/adds/deletes collections), the IDB must adapt automatically. No manual migrations. No version files. Hash-based drift detection.

## Goal

Implement the diff engine that compares expected stores (from model) vs actual IDB stores.

// machineIdbAdapter.ts

export function computeSchemaHash(storeNames: string[]): string {
  return [...storeNames].sort().join(',');
}

export async function getCurrentIdbStores(dbName: string): Promise<Set<string>> {
  return new Promise((resolve, reject) => {
    const req = indexedDB.open(dbName);
    req.onsuccess = (e) => {
      const db     = (e.target as IDBOpenDBRequest).result;
      const stores = new Set(Array.from(db.objectStoreNames));
      db.close();
      resolve(stores);
    };
    req.onerror = () => reject(req.error);
  });
}

## Acceptance

- `computeSchemaHash(['vehicle','appuser_prefs'])` → deterministic string (sorted join)
- `getCurrentIdbStores('testdb')` → Set of current store names (excludes `__*` internal stores)
- `getStoredSchemaHash` → null if no hash stored yet
- `storeSchemaHash` → persists hash in `__schema_meta__` store (auto-created if missing)
- All functions work in browser IndexedDB context
