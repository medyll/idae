# S16-02 — CRUD audit trail via AuditService

**Sprint:** 16  
**Effort:** S  
**Priority:** High  
**Deps:** S16-01

## Problem

`AuditService.logAudit()` existe (`server/src/services/AuditService.ts`) mais utilisé seulement pour auth events (login/logout/permission_denied).  
CRUD create/update/delete pas loggués → pas de traçabilité métier.  
Legacy `set_log()` loggait CREATE/UPDATE/DELETE sur chaque opération.

## What to do

### 1. Wire logAudit dans handlers data.ts

```ts
// Dans createRecord, après Model.create():
logAudit({
  action: 'create',
  userId: (req as any).user?.id,
  login: (req as any).user?.login,
  resourceType: table,
  resourceId: String(record._id),
  status: 'success',
  ...extractAuditContext(req)
});

// Dans updateRecord, après findByIdAndUpdate():
logAudit({
  action: 'update',
  userId: (req as any).user?.id,
  resourceType: table,
  resourceId: id,
  status: 'success',
  details: { fields: Object.keys(req.body) },
  ...extractAuditContext(req)
});

// Dans deleteRecord (soft), après update:
logAudit({
  action: 'delete',
  userId: (req as any).user?.id,
  resourceType: table,
  resourceId: id,
  status: 'success',
  details: { permanent: false },
  ...extractAuditContext(req)
});
```

### 2. Pas de nouvelle classe

ADR-01 : MachineHistory/MachineActivity sont anti-patterns (wrappers inutiles sur collection générique).  
`logAudit()` fire-and-forget → écrit dans `appuser_audit` (déjà dans AuditService).  
Si besoin accès côté client → `machine.collection('_history')` via idae-idbql sync.

### 3. userId depuis req.user

`requireDroit` middleware injecte le user. Vérifier shape dans `permission.ts` et typer correctement si besoin.

## Acceptance criteria

- [ ] POST `/api/data/vehicule` → entrée dans `appuser_audit` avec action='create', resourceType='vehicule'
- [ ] PUT `/api/data/vehicule/:id` → entrée action='update' avec `details.fields`
- [ ] DELETE `/api/data/vehicule/:id` → entrée action='delete' avec `details.permanent`
- [ ] `logAudit` fire-and-forget — échec d'écriture ne bloque pas la réponse
- [ ] userId présent si user authentifié
- [ ] Pas de nouveau service/classe créé
