# S52-02 ‚Äî BL-15: Waffle / start overlay (type‚Üícollection launch + launch_all)

## Story

Build the legacy waffle / start overlay (`app_gui_start_menu.php` / `menu.production` screen). It opens from a TaskBar waffle button and shows columns per `appscheme_type`, each listing permitted+preferred collections. Launch a single collection or `launch_all` for a whole type.

## Files

- New: `src/lib/shell/frame/start/StartMenu.svelte` (or `src/lib/shell/layout/WaffleMenu.svelte`)
- Modified: `src/lib/shell/layout/TaskBar.svelte` (add waffle toggle button)
- Registry: add `'start'` frame entry if frame-loaded; otherwise keep as layout overlay.
- Export: `src/lib/index.ts`

## Acceptance criteria

1. Waffle button in `TaskBar.svelte` toggles the overlay.
2. Overlay displays one column per `appscheme_type`.
3. Each column lists collections filtered by `filterMenuCollections('start', ...)` from `menuPrefs.ts`.
4. Dev mode bypasses pref filtering; empty prefs show all permitted collections.
5. Single collection launch fires the appropriate BL-13 framer verb (Espace/Cr√©er, etc.).
6. `launch_all` for a type iterates that type's nodes and opens each (e.g. multiple explorers).
7. No inline rights/prefs joins ‚Äî consume `useMenuTree('start')` and `menuPrefs.ts`.
8. No `machine.collection()` reads ‚Äî use `machine.store`.
9. Custom tags have explicit `display` CSS.
10. Component test verifies overlay open/close, column rendering, and launch verb wiring.
11. `pnpm run check` 0 errors, `pnpm run test` green.
