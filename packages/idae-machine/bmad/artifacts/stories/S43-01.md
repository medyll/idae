# S43-01 — DataRecord mode="row" + svelte:element — conformité ADR-06

> Sprint 43 | Effort S | Source ADR-06 (rectification S42-01)
> Status: ready

## Contexte

S42-01 a implémenté `mode='table'` avec un loop `DataField` directement dans `DataList.svelte`.
C'est une violation de l'architecture : **DataList ne doit pas boucler sur les fields**.
`DataRecord` est le seul composant autorisé à itérer les champs d'un record.

ADR-06 spécifie également `<svelte:element this={tag}>` pour les items — non implémenté.

**Invariant violé :** DataList contient `{#each tableColumns as col}<DataField .../>{/each}` dans son template.
**Directive non suivie :** `svelte:element` absent du rendu items.

## Variables à connaître (état actuel)

```
DataList.svelte
  - import DataField (ligne ~33) ← à supprimer
  - tableColumns $derived (ligne ~164) ← garder pour <thead> uniquement
  - {#snippet renderItem} (ligne ~311) ← refactorer avec svelte:element
  - table <tbody> loop (ligne ~396) ← remplacer field loop par DataRecord mode="row"

DataRecord.svelte
  - mode?: 'show' | 'create' | 'update' ← ajouter 'row'
  - fieldNames $derived (ligne ~63) ← réutiliser tel quel
  - effectiveData $derived (ligne ~60) ← réutiliser tel quel
```

## Changements à apporter

### 1. DataRecord.svelte — ajouter mode="row"

Ajouter `'row'` au type `mode`. Ajouter branche **avant** le `{#if groups}` existant :

```svelte
{#if mode === 'row'}
    {#if scheme && fieldNames.length && effectiveData != null}
        {#each fieldNames as fieldName (fieldName)}
            {#if scheme.fields?.[fieldName] && fieldName in effectiveData}
                <td>
                    <DataField {collection} {fieldName} mode="show" data={effectiveData} showLabel={false} />
                </td>
            {/if}
        {/each}
    {/if}
{:else if groups}
    ...existant inchangé...
```

Type à mettre à jour :
```ts
mode?: 'show' | 'create' | 'update' | 'row';
```

> **Contrainte :** mode="row" ne supporte pas `groupBy` (sémantiquement incohérent dans un `<tr>`).
> Si `mode === 'row'` et `groupBy` fourni → ignorer groupBy silencieusement (fieldNames non groupés).

### 2. DataList.svelte — refactorer renderItem avec svelte:element

**Avant (ligne ~311) :**
```svelte
{#snippet renderItem(record: COL, idx: number)}
    {#if itemSnippet}
        {@render itemSnippet({ record, idx, fieldValues })}
    {:else}
        {@const rec = record as Record<string, unknown>}
        {@const label = renderPresentation(rec)}
        <li>
            <button type="button" class="data-list-link"
                onclick={() => parsedLink ? navigate(rec) : handleItemClick(record)}>
                {#if label}{label}{:else}<DataRecord {collection} data={rec} mode="show" />{/if}
            </button>
        </li>
    {/if}
{/snippet}
```

**Après :**
```svelte
{#snippet renderItem(record: COL, idx: number)}
    <svelte:element
        this={currentMode === 'table' ? 'tr' : 'li'}
        class:clickable={currentMode === 'table' && (!!onItemClick || !!parsedLink)}
        onclick={currentMode === 'table' ? () => handleItemClick(record) : undefined}
    >
        {#if currentMode === 'table'}
            <DataRecord {collection} data={record as Record<string, any>} mode="row" {view} />
        {:else if itemSnippet}
            {@render itemSnippet({ record, idx, fieldValues })}
        {:else}
            {@const rec = record as Record<string, unknown>}
            {@const label = renderPresentation(rec)}
            <button type="button" class="data-list-link"
                onclick={() => parsedLink ? navigate(rec) : handleItemClick(record)}>
                {#if label}{label}{:else}<DataRecord {collection} data={rec} mode="show" />{/if}
            </button>
        {/if}
    </svelte:element>
{/snippet}
```

### 3. DataList.svelte — table <tbody> utilise renderItem

**Avant (ligne ~395) :**
```svelte
<tbody>
    {#each paginatedItems as record ((record as Record<string, unknown>)[indexField])}
        <tr
            class:clickable={!!onItemClick || !!parsedLink}
            onclick={() => handleItemClick(record)}
        >
            {#each tableColumns as col (col.name)}
                <td>
                    <DataField
                        {collection}
                        fieldName={col.name}
                        mode="show"
                        data={record as Record<string, unknown>}
                        showLabel={false}
                    />
                </td>
            {/each}
        </tr>
    {/each}
    ...
</tbody>
```

**Après :**
```svelte
<tbody>
    {#each paginatedItems as record, idx ((record as Record<string, unknown>)[indexField])}
        {@render renderItem(record as COL, idx)}
    {/each}
    {#if !paginatedItems.length}
        <tr><td colspan={tableColumns.length}>
            {#if emptySnippet}{@render emptySnippet()}{:else}—{/if}
        </td></tr>
    {/if}
</tbody>
```

### 4. DataList.svelte — supprimer import DataField

```ts
// Supprimer :
import DataField from '$lib/data-ui/field/DataField.svelte';
```

DataField n'est plus utilisé directement dans DataList après ce refactor.

### 5. DataList.svelte — mettre à jour JSDoc snippet item

```
@snippet item({ record, idx, fieldValues }) - Rendu custom par record (list + grid uniquement).
  Ignoré en mode table : le tableau délègue à DataRecord mode="row" (layout structurel).
```

## Ce qu'il NE FAUT PAS faire

| ❌ Mauvais | ✅ Correct |
|---|---|
| Garder `{#each tableColumns} <DataField/>` dans DataList | DataRecord mode="row" délègue le loop |
| Supprimer `tableColumns` | Le garder pour `<thead>` uniquement |
| Mettre groupBy dans mode="row" | Ignorer silencieusement |
| Ajouter `svelte:element` au container `<table>/<ul>` | Structures trop différentes — `{#if mode==='table'}` pour le container |
| Créer un composant DataTableRow séparé | Tout dans DataRecord mode="row" |
