# S16-01 — Soft delete : deletedAt + filter

**Sprint:** 16  
**Effort:** S  
**Priority:** High  
**Deps:** —

## Problem

`deleteRecord` fait un hard delete (`findByIdAndDelete`). Aucune trace, pas de recovery possible.  
Legacy PHP copiait dans `sitebase_app.trash` avant suppression — trop lourd.  
Approche moderne : champ `deletedAt` + filtre automatique.

## What to do

### 1. `deleteRecord` — soft delete

```ts
// server/src/routes/data.ts
export async function deleteRecord(req, res) {
  const { table, id } = req.params;
  const permanent = req.query.permanent === 'true';

  const Model = await getCollectionModel(table);

  if (permanent) {
    // Admin only — hard delete (requireDroit('D') + 'ADMIN' check à ajouter plus tard)
    const record = await Model.findByIdAndDelete(id).lean();
    if (!record) { res.status(404).json({ error: `Record '${id}' not found` }); return; }
    broadcastToTable(table, 'data:deleted', { id });
    res.status(204).send();
  } else {
    // Soft delete
    const record = await Model.findByIdAndUpdate(
      id,
      { deletedAt: new Date().toISOString() },
      { new: true }
    ).lean();
    if (!record) { res.status(404).json({ error: `Record '${id}' not found` }); return; }
    broadcastToTable(table, 'data:deleted', { id });
    res.status(204).send();
  }
}
```
