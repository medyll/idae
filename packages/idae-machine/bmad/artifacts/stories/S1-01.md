## User Story

As a **developer working on idae-machine**, I want to **replace 42+ `any` type instances with strict generics**, so that **type safety is enforced at compile time and runtime errors are caught early**.

---

## Context

The baseline audit identified 42+ `any` instances across core modules (`CrudService.ts`, `MachineSchemeValidate.ts`, `MachineSchemeFieldValues.ts`). This defeats TypeScript's type safety during form manipulation and validation.

**Linked Audit Finding**: "Type Safety: 42+ `any` Instances" (critical)  
**Linked PRD Section**: Type Safety Requirements (Tech Spec)  
**Linked PR Issue**: None (new work)

## Technical Notes

### Files to Update (Priority Order)

1. **`src/lib/db/CrudService.ts`** (25 lines, ~5 `any` instances)
   - Replace `Record<string, any[]>` with generic `Record<K, V[]>`
   - Type parameters in methods

2. **`src/lib/form/types.ts`** (40 lines, ~8 `any` instances)
   - Type all component props interfaces
   - Use `Record<string, unknown>` instead of `any`

3. **`src/lib/main/machine/MachineSchemeValidate.ts`** (80+ lines, ~6 `any` instances)
   - Type form data parameter: `formData: Record<string, unknown>`
   - Type validate method return

4. **`src/lib/main/machine/MachineSchemeFieldValues.ts`** (50+ lines, ~10 `any` instances)
   - Generic type parameter for field values
   - Type `format()` return type precisely

5. **`src/lib/main/machine/MachineSchemeValues.ts`** (60+ lines, ~8 `any` instances)
   - Generic row type parameter
   - Type collection utilities

### Refactoring Pattern

```typescript
// BEFORE
export class MachineSchemeValues {
  format(fieldName: string, value: any): string | string[] | any[] {
    // loss of type info
  }
}

// AFTER
export class MachineSchemeValues<T extends Record<string, unknown>> {
  format<K extends keyof T>(fieldName: K, value: T[K]): string {
    // full type safety
  }
}
```

### Testing Strategy

1. No new tests required (refactor existing code)
2. Run existing test suite: `pnpm run test` (must pass 100%)
3. Type check: `pnpm run check` (must have 0 errors)
4. Manual inspection of 5 updated files for accuracy

### Out of Scope

- Async validator types (deferred to S1-02)
- Component props types (minor; S1-01 focuses on logic layer)
- Third-party library type stubs (use `@ts-expect-error` if needed)
