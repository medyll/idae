# Story S1-01: Set up idae-api server with MongoDB

**Sprint:** S1 — Foundation  
**Status:** Ready for Development  
**Priority:** High  
**Estimated Effort:** 2-3 days

---

## User Story

**As a** developer  
**I want** an idae-api server running with MongoDB  
**So that** I can serve schema definitions and handle API requests

---

## Acceptance Criteria

- [ ] Server starts on port 3000
- [ ] MongoDB connection established
- [ ] `/health` returns 200 with version
- [ ] Error middleware catches 500s and returns JSON
- [ ] CORS allows requests from localhost:5173

---

## Technical Requirements

### Server Setup
- Express server with idae-api
- MongoDB connection via Mongoose
- Health check endpoint
- Error handling middleware
- CORS configuration

### File Structure
```
server/
├── src/
│   ├── index.ts              # Entry point
│   ├── config.ts             # Environment config
│   ├── app.ts                # Express app setup
│   ├── routes/
│   │   └── health.ts
│   ├── middleware/
│   │   ├── errorHandler.ts
│   │   └── cors.ts
│   └── models/
│       └── AppScheme.ts      # MongoDB schema
├── package.json
└── tsconfig.json
```

### API Endpoints
```
GET /health          → { status: 'ok', version: '2.0.0' }
```
