# S28-03 — machine.adaptIdbToSchema() — wire drift detection into start()

**Effort:** M
**Priority:** high

## Goal

Wire the drift detection into `machine.start()`. Called automatically before `createQoolie()`. If drift detected, upgrades IDB first, then continues with normal start.

## Files

| File | Change |
|------|--------|
| `src/lib/main/machine.ts` | `adaptIdbToSchema()` + wire into `start()` |

## Implementation

```ts
// machine.ts

private async adaptIdbToSchema(): Promise<void> {
  if (!this._dbName || !this._effectiveModel) return;

  const expectedStores = Object.entries(this._effectiveModel).map(([name, col]) => ({
    name,
    keyPath: col.keyPath,
  }));
  const expectedNames = new Set(expectedStores.map(s => s.name));
  const schemaHash    = computeSchemaHash([...expectedNames]);

  const storedHash    = await getStoredSchemaHash(this._dbName).catch(() => null);
  if (storedHash === schemaHash) return; // no drift

  const currentStores = await getCurrentIdbStores(this._dbName).catch(() => new Set<string>());

  const INTERNAL = /^__/; // protect __outbox__, __schema_meta__, __migrations__

  const toCreate = expectedStores.filter(s => !currentStores.has(s.name));
  const toDelete = [...currentStores].filter(s => !expectedNames.has(s) && !INTERNAL.test(s));

  if (toCreate.length || toDelete.length) {
    const currentVersion = await getCurrentIdbVersion(this._dbName).catch(() => 1);
    await upgradeIdb(this._dbName, toCreate, toDelete, currentVersion);
  }

  await storeSchemaHash(this._dbName, schemaHash).catch(() => {});
}

// start() — updated
async start(): Promise<void> {         // becomes async
  this._effectiveModel = this.buildEffectiveModel();
  this.createCollections();
  await this.adaptIdbToSchema();       // drift check BEFORE createStore
  this.createStore();
  this.loadPolicies();
  if (this._socketOptions?.autoConnect) this.connectSocket();
}
```

## Key invariant

`adaptIdbToSchema()` runs BEFORE `createStore()` (which calls `createQoolie()`). Qoolie opens the IDB — it must find the correct stores already in place.

## Acceptance

- No drift → start() proceeds normally, no IDB open/close overhead
- Drift detected → IDB upgraded → createQoolie sees correct stores
- `start()` signature change: `void` → `Promise<void>` (callers must await)
- `machine.start()` callers in tests and layout updated accordingly
