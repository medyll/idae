# Story S7-01 — seedSchemeFromModel + CLI + POST /api/bootstrap

**Sprint:** 7  
**Goal:** Bootstrap MongoDB appscheme_* from IdbqModel

## Implemented

| File | Action |
|------|--------|
| `server/src/bootstrap/seedSchemeFromModel.ts` | NEW — core seeder, idempotent upserts |
| `server/src/bootstrap/seed.ts` | NEW — CLI entry (`tsx server/src/bootstrap/seed.ts [org] [mongoUri]`) |
| `server/src/routes/bootstrap.ts` | NEW — POST /api/bootstrap handler |
| `server/src/index.ts` | MOD — registers bootstrap route in development only |

## Behavior

- `seedSchemeFromModel(model, { org, mongoUri })`:
  - Upserts `appscheme_base` for each unique `base` value
  - Upserts `appscheme` per collection (code, base, index, presentation, keyPath)
  - Upserts `appscheme_field` per field (collection, name, type, required, readonly, private)
  - Target DB: `{org}_machine_app` via `mongoose.connection.useDb()`
  - Idempotent — safe to run multiple times
- `POST /api/bootstrap`: dev-only, body `{ org, model }`, calls seedSchemeFromModel
- CLI: `tsx server/src/bootstrap/seed.ts test mongodb://localhost:27017`
