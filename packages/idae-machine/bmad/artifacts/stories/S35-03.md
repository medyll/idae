# Story S35-03 — bulkUpsertSilent(records) on IdbCollection

## Context
Part of QOOLIE-SWR sprint (S35). The HydrationController (S35-01) needs a way to write server-pulled records into IndexedDB without triggering outbox sync events, while still notifying reactive adapters that data has changed.

## Acceptance Criteria

1. **Method signature**: `bulkUpsertSilent(items: Partial<T>[]): Promise<T[]>` added to `IdbCollection<T>`.
2. **Write semantics**: Each record written via IDB `put()` (upsert — overwrites if key exists, inserts if not).
3. **No outbox**: Does NOT add to outbox / sync layer. Uses existing `{ silent: true }` internal write path.
4. **Reactive notification**: After batch completes, fires `idbEventBus 'change'` event so `useQoolieCollection` re-renders with hydrated data.
5. **Transaction safety**: All writes occur in a single IndexedDB transaction (same pattern as `batchPut`).
6. **Return value**: Returns array of written records with their keys populated.

## Implementation Notes

- Internally delegates to `batchPut(items, { silent: true })` to reuse the existing write queue and transaction logic.
- Then explicitly calls `this.eventBus.emit(this._store, 'put', results)` to trigger reactive updates.
- This avoids duplicating the write stack while achieving the "silent to sync, loud to reactivity" requirement.

## Tests

- `IdbCollection.test.ts` add `describe('bulkUpsertSilent')` with cases:
  - writes multiple records
  - overwrites existing records (upsert)
  - emits 'change' event on event bus
  - does NOT emit standard 'put' event through the normal channel (verify via bus.on with { silent: true } behavior)
  - uses single transaction (all-or-nothing)

## Dependencies
- None — foundational story. S35-01 depends on this.
