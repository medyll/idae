# S28-02 — IDB upgrade: create missing stores + delete orphans

**Sprint:** 28
**Effort:** M
**Priority:** high
**Dependencies:** S28-01

---

## Goal

Execute the actual IDB schema upgrade when drift is detected: open IDB at version+1, create missing stores in `onupgradeneeded`, delete orphaned stores.

## Files

| File | Change |
|------|--------|
| `src/lib/main/machineIdbAdapter.ts` | Add `upgradeIdb()` |

## Implementation

```ts
export async function upgradeIdb(
  dbName:   string,
  toCreate: { name: string; keyPath: string }[],
  toDelete: string[],
  currentVersion: number,
): Promise<void> {
  return new Promise((resolve, reject) => {
    const req = indexedDB.open(dbName, currentVersion + 1);

    req.onupgradeneeded = (e) => {
      const db = (e.target as IDBOpenDBRequest).result;

      for (const store of toDelete) {
        if (db.objectStoreNames.contains(store)) {
          db.deleteObjectStore(store);
        }
      }

      for (const { name, keyPath } of toCreate) {
        if (!db.objectStoreNames.contains(name)) {
          const autoIncrement = keyPath.startsWith('++');
          const realKey       = keyPath.replace('++', '');
          db.createObjectStore(name, { keyPath: realKey, autoIncrement });
        }
      }
    };

    req.onsuccess = () => { (req.result).close(); resolve(); };
    req.onerror   = () => reject(req.error);
  });
}
```

## Acceptance

- New stores created with correct keyPath and autoIncrement
- Orphaned stores deleted (no data preservation — server sync will repopulate)
- `__*` internal stores (qoolie: `__outbox__`, `__schema_meta__`) never deleted
- IDB version incremented by 1 on each upgrade
- Qoolie reconnects to the upgraded IDB after (machine.start() re-runs createQoolie)
