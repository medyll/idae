# S18-04 — initialSortBy prop on ExplorerTable

**Sprint:** 18  
**Effort:** S  
**Priority:** medium  
**Dependencies:** S18-01 (SortBy type in explorerUtils.ts)

---

## Context

`ExplorerTable` already has full internal sort state (`sortColumn: $state`, `sortDirection: $state`)
and click-to-sort headers (S14-03, `explorerTableSort.test.ts`).

Problem: the table always starts unsorted. Callers cannot preset the initial sort without
duplicating logic outside the component.

This story adds `initialSortBy` — an uncontrolled initial value. After mount, click-to-sort
continues to own the state (no external control). Named `initialSortBy` (not `sortBy`) to
signal it is read once at init, not a controlled binding.

## ExplorerTable change

```diff
+ import type { SortBy } from './explorerUtils.js';

- let { items = [], columns = [], sortable = false, sortableColumns = [] } = $props();
+ let {
+   items = [],
+   columns = [],
+   sortable = false,
+   sortableColumns = [],
+   initialSortBy,
+ }: {
+   items?: Record<string, unknown>[];
+   columns?: string[];
+   sortable?: boolean;
+   sortableColumns?: string[];
+   initialSortBy?: SortBy;
+ } = $props();

- let sortColumn    = $state<string | null>(null);
- let sortDirection = $state<'asc' | 'desc'>('asc');
+ let sortColumn    = $state<string | null>(initialSortBy?.field ?? null);
+ let sortDirection = $state<'asc' | 'desc'>(initialSortBy?.direction ?? 'asc');
```

No other changes. `sortedItems` derived, `toggleSort`, `sortIndicator` — all unchanged.

---

## Acceptance criteria

1. `<ExplorerTable {items} columns={['name','age']} sortable initialSortBy={{ field: 'name', direction: 'asc' }}>` → table renders pre-sorted by name asc on mount
2. Clicking a column header still toggles sort (internal state, not re-read from prop)
3. `initialSortBy={{ field: 'age', direction: 'desc' }}` → table starts sorted by age desc
4. Omitting `initialSortBy` → state starts as `null`/`'asc'` — identical to current behavior
5. `initialSortBy.field` not in `columns` → no crash, table unsorted (same as clicking an unknown column: `effectiveSortable.includes(col)` guard already handles it)
