# S27-02 — Explorer.svelte consomme _views par mode

**Sprint:** 27
**Effort:** M
**Priority:** high
**Dependencies:** S27-01, S25-02 (Explorer.svelte exist)

---

## Context

Explorer.svelte (S25-02) utilise `template.presentation` pour colonnes et champs. S27-01 rend `_views` disponible. Ce story câble `_views` dans Explorer.

## File

| File | Change |
|------|--------|
| `src/lib/shell/explorer/Explorer.svelte` | Consommer `scheme.views` par mode |

## Logic

```ts
const scheme = $derived(machine.logic.collection(collection));
const views  = $derived(scheme?.views ?? {});  // MachineScheme.views getter (S27-03)

const listFields = $derived(
  views.list?.map(f => f.name)
  ?? scheme?.template?.presentation?.split(' ')
  ?? []
);

const formFields = $derived(
  views.form?.map(f => f.name) ?? undefined  // undefined = CardForm montre tout
);

const miniLabel = $derived(
  views.mini?.[0]?.name ?? listFields[0] ?? 'id'
);

const fkLabel = $derived(
  views.fk_label?.map(f => f.name) ?? listFields.slice(0, 2)
);
```

## Dispatch

- `mode='list'` → itère `listFields` pour colonnes de chaque row
- `mode='table'` → `listFields` pour headers + cells
- `mode='card'` → passe `formFields` à CardForm prop `showFields` (si undefined = tout)
- `mode='actions'` → utilise `miniLabel` pour texte du bouton

## Acceptance

- Si `_views` absent → fallback `template.presentation` silencieux, 0 crash
- Si `_views` présent → colonnes/champs correspondent aux vues définies
- Svelte 5 runes uniquement
