# Story S3-01: Server Broadcast on CRUD operations

**Sprint:** S3 — Real-Time  
**Status:** Ready for Development  
**Priority:** High  
**Estimated Effort:** 2 days

---

## User Story

**As a** connected client  
**I want** to receive real-time updates when data changes  
**So that** my UI stays in sync with other users

---

## Acceptance Criteria

- [ ] Socket.IO server initialized on port 4000
- [ ] Broadcast 'create' event after POST
- [ ] Broadcast 'update' event after PUT
- [ ] Broadcast 'delete' event after DELETE
- [ ] Room per table: `room_{table}`
- [ ] JWT authentication for socket connections
- [ ] Throttling to prevent flood

---

## Technical Requirements

### Server Setup

```typescript
import { SocketIoServer } from '@medyll/idae-socket';

const socketServer = new SocketIoServer({
  port: 4000,
  auth: {
    strategy: 'jwt',
    jwtSecret: process.env.JWT_SECRET
  }
});

socketServer.init();
```

### Broadcast on CRUD

```typescript
// After successful CRUD operation
io.to(`room_${table}`).emit('data:created', { table, record });
io.to(`room_${table}`).emit('data:updated', { table, id, record });
io.to(`room_${table}`).emit('data:deleted', { table, id });
```
