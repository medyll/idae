# Story S2-01: Implement CRUD endpoints with pagination

**Sprint:** S2 — Data Layer  
**Status:** Ready for Development  
**Priority:** High  
**Estimated Effort:** 3-4 days

---

## User Story

**As a** developer  
**I want** CRUD endpoints for data operations  
**So that** I can create, read, update, and delete records via the API

---

## Acceptance Criteria

- [ ] `GET /api/data/:table` — List records with pagination
- [ ] `GET /api/data/:table/:id` — Get single record
- [ ] `POST /api/data/:table` — Create new record
- [ ] `PUT /api/data/:table/:id` — Update existing record
- [ ] `DELETE /api/data/:table/:id` — Delete record
- [ ] Pagination: `?page=1&limit=20` query params
- [ ] Sorting: `?sort=field&order=asc|desc`
- [ ] Filtering: `?filter[field]=value`
- [ ] Response includes metadata: total, page, pages

---

## Technical Requirements

### API Endpoints

```
GET    /api/data/:table          → List with pagination
GET    /api/data/:table/:id      → Single record
POST   /api/data/:table          → Create
PUT    /api/data/:table/:id      → Update
DELETE /api/data/:table/:id      → Delete
```

### Response Format

```json
// List response
{
  "data": [...],
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "pages": 5
  }
}

// Single record
{
  "id": "123",
  "name": "Product",
  ...
}
```
