# Story S8-02 — data.ts multi-DB routing via appscheme

**Sprint:** 8  
**Goal:** Route /api/data/:table to correct MongoDB database per collection's `base`

## Implemented

| File | Action |
|------|--------|
| `server/src/middleware/dbRouter.ts` | NEW — getDbForCollection(), caches base lookups |
| `server/src/routes/data.ts` | MOD — uses getDbForCollection() for all CRUD ops |

## Behavior

- `getDbForCollection(collectionName)`:
  - Looks up `appscheme` in `{org}_machine_app` to find `base` for the collection
  - Caches result (module-level Map) — one lookup per collection per process lifetime
  - Returns `mongoose.connection.useDb('{org}_{base}', { useCache: true })`
  - Falls back to `machine_base` if not found in appscheme
