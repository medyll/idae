## Problem

idae-machine n'a pas de convention pour la logique métier custom par domaine.  
Legacy PHP avait `mdl/app/{module}/actions.php` colocalisé — bonne idée, mauvaise implémentation (dispatch F_action, pas de typage, pas d'erreur handling).  
Besoin : même localité, mais TypeScript, Mongoose hooks, composable.

## Decision: Option A — colocalisé avec le schéma

```
server/src/models/
  demo/
    demoScheme.ts        ← schéma (existe déjà)
    actions.ts           ← hooks + logique métier domain ← NOUVEAU
```

Pas de répertoire `extensions/` séparé. Jamais loin du schéma.

## What to do

### 1. Établir le contrat `DomainActions`

```ts
// server/src/models/domainActions.ts  ← interface partagée
export interface DomainActions {
  /** Mongoose schema hooks ou Express middleware pre/post */
  registerHooks?: (schema: Schema, collection: string) => void;
  /** Validation extra (après MachineSchemeValidate) */
  validate?: (data: unknown, collection: string) => { valid: boolean; errors: Record<string, string> };
  /** Logique post-create custom */
  afterCreate?: (record: unknown, collection: string) => Promise<void>;
  /** Logique post-update custom */
  afterUpdate?: (record: unknown, collection: string) => Promise<void>;
  /** Logique pre-delete custom */
  beforeDelete?: (id: string, collection: string) => Promise<void>;
}

/** Registry global — chargé au boot */
export const domainActionsRegistry = new Map<string, DomainActions>();

export function registerDomainActions(collection: string, actions: DomainActions): void {
  domainActionsRegistry.set(collection, actions);
}
```

### 2. Demo — rental/vehicule domain actions

```ts
// server/src/models/demo/actions.ts
import { registerDomainActions } from '../domainActions.js';

registerDomainActions('vehicule', {
  validate(data) {
    const errors: Record<string, string> = {};
    const d = data as Record<string, unknown>;
    if (d.kilometrage && Number(d.kilometrage) < 0)
      errors.kilometrage = 'Kilométrage ne peut pas être négatif';
    if (d.prixJour && Number(d.prixJour) <= 0)
      errors.prixJour = 'Prix journalier doit être > 0';
    return { valid: Object.keys(errors).length === 0, errors };
```

### 3. Brancher le registry dans data.ts handlers

```ts
// Dans createRecord, après Model.create():
const domainActions = domainActionsRegistry.get(table);
if (domainActions?.afterCreate) {
  void domainActions.afterCreate(record, table).catch(err => logger.error('afterCreate hook failed:', err));
}

// Dans deleteRecord, avant soft-delete:
const domainActions = domainActionsRegistry.get(table);
if (domainActions?.beforeDelete) {
  await domainActions.beforeDelete(id, table);
}
```

### 4. Charger les domain actions au boot

```ts
// server/src/index.ts ou MachineServer.ts — au boot:
import './models/demo/actions.js';  // charge + enregistre
```
