## Findings

### 🔴 URGENT

**1. Duplicate exports — index.ts:45-47**
- `CollectionCard` and `CollectionTable` exported twice: once from `$lib/demo/`, once from `$lib/ui/`
- Last export wins silently — creates namespace ambiguity and may ship demo components as public API
- **Fix:** Remove the demo exports; keep only `$lib/ui/` variants

---

### 🟠 HIGH

**2. `any` type in public API — src/lib/demo/demoInit.ts:5**
- `idbql: any` parameter — should use the proper IdbQL type
- **Fix:** Import and use `IdbqlDb` or equivalent type from `@medyll/idae-idbql`

---

### 🟡 MEDIUM

**3. Missing keys in `{#each}` — CLAUDE.md rule violation**
- `src/lib/collection/CollectionList.svelte:13` — `{#each collections as collection}` → add `(collection.code)`
- `src/lib/data/DataListFields.svelte:46` — `{#each fieldNames() as fieldName}` → add `(fieldName)`

**4. Silent catches**
- `src/lib/data/DataList.svelte:49` — catch returns `null as any` → throw `MachineError` instead
- `src/lib/demo/demoInit.ts` — catch with only `console.error` → acceptable for demo, note it

**5. `any` types (8 occurrences)**
- `DataLinks/DataLinksBack`: `collectionId?: any` → `string | number`
- `DataList/DataListActions`: `id: any` in `openCrud()` → `string | number`
- `CollectionCard/CollectionTable` (demo): `items: any[]` → `Record<string, unknown>[]`
- `InfoLine`, `Selector`: `value?: any`, `values?: any[]` → `unknown` or typed union

---

## Recommended Fix Order

1. Fix duplicate exports in `src/lib/index.ts` (prevents incorrect public API surface)
2. Type `idbql` param in `demoInit.ts`
3. Add `{#each}` keys to `CollectionList` and `DataListFields`
4. Replace `null as any` in `DataList.svelte:49` with `MachineError`
5. Replace `any` types across DataLinks, DataList, InfoLine, Selector
