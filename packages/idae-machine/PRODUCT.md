# Product

## Register

product

## Users

Developers and technical builders who define, inspect, and evolve schema-driven CRUD applications. They work in a technical context: local development, architecture review, debugging, and product assembly across Svelte, IndexedDB, MongoDB, sync, and RBAC surfaces.

## Product Purpose

`@medyll/idae-machine` lets a team declare a schema once, deploy it to a server-owned meta-schema, boot a local reactive runtime from that schema, and obtain generic CRUD UI, validation, FK/RFK relations, frame-based navigation, offline-first storage, and sync without rewriting each screen by hand.

Success means the system feels coherent end to end: schema authoring, deployment, runtime boot, storage, and UI all read as one machine rather than a pile of disconnected subsystems.

## Brand Personality

Systematic, expert, structural.

The tone should feel precise, confident, and architecture-first. It should help a technical reader build a clear mental model quickly, without noise, gimmicks, or decorative abstraction.

## Anti-references

- Generic SaaS marketing aesthetics applied to technical product surfaces
- Over-decorated dashboards that hide structure behind cards and gradients
- Vague “AI tool” visual language that looks impressive but explains nothing
- Cute developer docs styling that reduces scanability or information density

## Design Principles

1. Show the chain, not isolated pieces.
2. Make system boundaries visible: schema, server, runtime, storage, UI.
3. Favor structural clarity over decorative intensity.
4. Let the product feel rigorous, not corporate-polished or marketing-shaped.
5. Keep technical density readable: compact, but never cramped or muddy.

## Accessibility & Inclusion

Target WCAG AA contrast on all text and UI structure. Preserve keyboard-readable hierarchy, visible focus states, reduced-motion-safe behavior, and long-form readability for dense technical content.
