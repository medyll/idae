# Sprint 1 — Core Stability & Query Foundation

**Goal:** Stabilize the core CRUD operations, fix known reactivity bugs, and establish a solid testing foundation before adding advanced features.

**Duration:** 1 week
**Capacity:** 5 stories (mixed effort)

---

## Stories

| ID | Title | Effort | Priority |
|----|-------|--------|----------|
| S1-01 | Fix transaction deadlocks on rapid writes | M | High |
| S1-02 | Fix inconsistent reactivity on multi-store updates | M | High |
| S1-03 | Add comprehensive unit tests for CollectionCore CRUD | L | High |
| S1-04 | Improve TypeScript type inference for nested models | M | Medium |
| S1-05 | Document schema migration patterns with examples | S | Low |

**Total Effort:** 3M + 1L + 1S

---

## Story Details

### S1-01 — Fix transaction deadlocks on rapid writes
**Effort:** M  
**Description:** Investigate and resolve occasional transaction deadlocks that occur during rapid write operations.  
**Acceptance Criteria:**
- [ ] Identify root cause (lock contention, missing await, transaction scope)
- [ ] Implement fix (queue writes, optimize transaction lifecycle, or add retry logic)
- [ ] Add stress test that reproduces rapid writes without deadlock
- [ ] All existing tests pass

**Dependencies:** None

---

### S1-02 — Fix inconsistent reactivity on multi-store updates
**Effort:** M  
**Description:** Ensure reactive state updates consistently propagate when multiple stores are modified in sequence.  
**Acceptance Criteria:**
- [ ] Reproduce the inconsistency scenario
- [ ] Fix event propagation in idbqlEvent
- [ ] Add test case for multi-store update reactivity
- [ ] Verify Svelte components react correctly to cross-store changes

**Dependencies:** None

---

### S1-03 — Add comprehensive unit tests for CollectionCore CRUD
**Effort:** L  
**Description:** Expand test coverage for all CollectionCore methods (add, put, get, getAll, delete, where).  
**Acceptance Criteria:**
- [ ] Test all CRUD operations with valid data
- [ ] Test edge cases (empty store, missing keys, duplicate keys)
- [ ] Test where() queries with various operators
- [ ] Achieve >80% coverage on collection.svelte.ts
- [ ] All tests run in CI without flakiness

**Dependencies:** None

---

### S1-04 — Improve TypeScript type inference for nested models
**Effort:** M  
**Description:** Enhance type inference so deeply nested model structures are correctly typed without explicit annotations.  
**Acceptance Criteria:**
- [ ] Review current type definitions in createIdbqDb and CollectionCore
- [ ] Implement generic type improvements
- [ ] Test with 3+ levels of nested objects
- [ ] Update TypeScript examples in docs

**Dependencies:** None

---

### S1-05 — Document schema migration patterns with examples
**Effort:** S  
**Description:** Create clear documentation and code examples for schema migrations (version upgrades, data transformation).  
**Acceptance Criteria:**
- [ ] Document the upgrade() callback pattern
- [ ] Provide 3 real-world migration examples (add index, rename field, transform data)
- [ ] Add to README.md or dedicated MIGRATIONS.md
- [ ] Include rollback guidance

**Dependencies:** None

---

## Dependencies Between Stories
- No critical path dependencies — all stories can be worked in parallel
- S1-03 (tests) will validate fixes from S1-01 and S1-02

## Risks
- S1-01 and S1-02 may require deep debugging — time-box investigation to 2h each before escalating
- S1-03 is the largest story — split into smaller test suites if needed

---

## Definition of Done (Sprint Level)
- [ ] All 5 stories marked complete with tests executed
- [ ] No regressions in existing functionality
- [ ] Test coverage increased from current baseline
- [ ] Documentation updated for migration patterns
- [ ] CHANGELOG.md updated with fixes and improvements
